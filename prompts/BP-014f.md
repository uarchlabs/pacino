### UAON mechanism summary

Two 4b saturating counters, uaon[0] and uaon[1], one per
prediction slot. Owned by tage_cntrl.

Trigger condition at predict time:
  prm CTR is weak (3'b011 or 3'b100) AND uaon[s] >= TAGE_UAON_THRES

When trigger fires: tage_using_primary=0, alt supplies
prediction. When trigger does not fire: tage_using_primary=1
(subject to prm CTR not being weak, or counter below
threshold).

UAON update rules (from tage_cntrl_uaon_update_rules.md):
  if pred_strong: no change
  if prm_wrong AND alt_correct: INC uaon[s]
  if prm_correct AND alt_wrong: DEC uaon[s]
  else: no change

pred_strong reflects NOT WEAK on the final provider CTR
after UAON mux selection.

TAGE_UAON_THRES = 8 (from bp_defines_pkg.sv).

### Provider role assignment

The primary provider is the longest-history tagged table
that hits. The alternate provider is the next longest-
history tagged table that hits below the primary. When
both T1 and T2 hit and T3/T4 do not, T2 is primary
(comp=2, 13b history) and T1 is alternate (comp=1, 8b
history).

Both TC-40 and TC-41 use T2 as primary and T1 as
alternate, both hitting. T3/T4 do not hit.

### Test sequence overview

TC-40 and TC-41 use the same PC and RAM entries.
TC-41 begins from the RAM state left by TC-40.
The two tests are sequenced in the main initial block
so TC-40 runs before TC-41.

### Address derivation notes

Formula (folded_hist=0 in all tests):
  idx_hash = (pc >> INST_OFFSET)[10:0] where INST_OFFSET=2
  bank = idx_hash[10]
  row  = idx_hash[9:0]
  tag_hash = (pc >> THIS_INDEX_BITS)[7:0] where
             THIS_INDEX_BITS=11

Tag formula verified from tage_table.sv lines 178-182:
  tag_hash = THIS_TAG_BITS'(
    (pc >> THIS_INDEX_BITS) ^ fh1_ext ^ (fh2_ext<<1))
  With fh=0: tag = (pc >> 11)[7:0]

T0 uses the same index formula as T1-T4 (pc[12:2]),
so bank and row are identical for T0 and T1 at a given PC.

## RTL Behavior vs Planning Document Notes

The UAON update in tage_cntrl.sv does NOT require
pred_diff to be set. The planning document confirms this.
The test uses pred_diff=1 (T2 and T1 predict opposite
directions) but UAON update would also fire for pred_diff=0
scenarios (tested indirectly).

HAND-FIX-001 (use_we_s0 = val & use_wr_u0 & prm_alt_match)
is gated by t_use_wr_u0 which is already table-specific in
tage_cntrl.sv (only asserted for the USE target table).
USE write fires only for the target table, not for both
prm and alt simultaneously. Verified from tage_cntrl.sv
lines 870-872.
