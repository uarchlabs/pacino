## Hypothesis
The IA-generated CTR and USE tests in tb_ittage.sv were
written against earlier draft versions of the planning
documents. The CTR table has been replaced with a fully
specified 33-row table. The USE table DIFF definition
was wrong (pred_tgt vs alt_tgt instead of prm_tgt vs
alt_tgt). Existing tests are likely invalid or incomplete.

## Background
ittage_cntrl_ctr_update_rules.md in session-045:
- Replaced TBD draft table with 33 fully specified rows.
- H=0 row (row 1): no update, no entry to write.
- UP=0 rows (2-17): only aACT active, pACT=--. pT and
  pCMP are don't care for CTR update but retained for
  test coverage. Each row maps to one test.
- UP=1 rows (18-33): only pACT active, aACT=--. aT and
  aCMP are don't care but retained for test coverage.
- Assert rows A1/A2/A3: impossible conditions enforced
  by ittage_assert.sv. No RTL action.
- MIS derived from PT != RT (not a stored signal).

ittage_cntrl_use_update_rules.md in session-045:
- DIFF corrected: ittage_prm_tgt != ittage_alt_tgt.
  Previous definition (pred_tgt vs alt_tgt) was wrong.
- HIT=0 row added as row 2 (no tagged hit, no update).
- Row suppression notes corrected.

Known bugs from BP-040 (B, C, D) in ittage_cntrl.sv
have not yet been independently verified. If a test
fails and the failure is consistent with Bug B, C, or D
as described in BP-040, stop and report in Results
Capture before attempting any fix. Do not modify
ittage_cntrl.sv without explicit instruction.

## Binding Previous Decisions
- Do not modify ittage_cntrl.sv or ittage.sv RTL unless
  a genuine RTL bug is found, explicitly cited in the
  planning documents. Stop and report before fixing.
- Tests must be independent of each other. Each test
  resets the relevant RAM entries before running.
- ITTAGE has no T0 base table. There are no BIM rows.
  Tests cover IT1-IT5 only.
- CTR is a confidence counter, not a direction counter.
  INC = higher confidence, DEC = lower confidence.
  MIS = indir_mispredict = (predicted_target !=
  resolved_target).

## Specific Requirements

2. For CTR tests: identify each test case in tb_ittage.sv
   that exercises CTR update rules. For each test:
   a. Map it to the corresponding row(s) in
      ittage_cntrl_ctr_update_rules.md.
   b. Verify the stimulus matches the row inputs.
   c. Verify the expected values match the row outputs.
   d. Mark as VALID, INVALID, or MISSING-COVERAGE.

   ittage_cntrl_use_update_rules.md.
