in g_ctr_upd." This session independently verifies that the

  ittage_using_primary=1 the block asserts alt_ctr_wr_u0

  ittage_cntrl_ctr_update_rules.md rows 18-33 specify

  instead of prm_ctr is wrong.

  alt_tbl_sel=0 (no alt entry) so no RAM write occurs.

  ittage_cntrl_ctr_update_rules.md. The planning doc is
  the authority. RTL must be corrected to match rows 18-33.
- Required fix (NOT applied here): in g_ctr_upd, swap the
  write strobe assignments so that using_primary=1 drives
  prm_ctr_wr_u0 (not alt_ctr_wr_u0) and using_primary=0
