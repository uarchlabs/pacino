## Hypothesis
sc_cntrl is the SC logic layer. On the prediction phase it forms the
signed sum of the five table counters plus the TAGE extended counter,
computes the local SC direction and confidence bands, applies the
two-corner chooser against TAGE confidence, and emits the final
direction and the sc_pred_meta capture. On the update phase it
applies the counter-update gate, adapts the dynamic threshold via the
TC counter, trains the chooser, and maintains the BrIMLI registers.
sc_cntrl holds all SC control state; the tables are pure RAM. The
control logic is fully specified in sc_decisions.md sections 8-12 and
can be implemented and unit-tested against driven stimulus.

## Assumptions made not explicit in the prompt
- Scalar SC control/BrIMLI state (sc_decisions.md section 8 declares
  threshold/TC/chooser/BrIMLI as scalars) is shared across both
  prediction slots. The scalar spec does not address dual-slot
  updates in the same cycle. Resolution: the lowest-indexed valid
  update slot drives the shared threshold/TC/chooser/BrIMLI
  adaptation; per-table counter writes proceed for every valid slot
  independently (each slot owns its per-slot table RAM, so there is
  no write conflict). Documented in the sc_cntrl header.
- tage_pred_meta_p2 and the table idx_hash_p2 outputs are registered
  one cycle (p2 -> p3) inside sc_cntrl so they align with ctr_p3 when
  the SC result forms at p3. sc_pred_meta_p3 is combinational at p3
  (reads ctr_p3 plus the staged/global FFs), which also satisfies the
  stl_sequent rule (the block reads FF outputs).
- The prediction-phase table read enable driven to the tables
  (t_sc_pred_val_p2) is tied to tage_pred_rdy_p2.
- br_imli saturating increment saturates at all-ones (10'h3FF).
- sc_upd_rdy_u1 is sc_upd_val_u0 registered one cycle (mirrors
  tage_upd_rdy_u1).
