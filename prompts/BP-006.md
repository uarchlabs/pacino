## Context Loaded
@rtl/bp/bp_defines_pkg.sv

## Hypothesis
A standalone combinational hash module isolates index and tag
computation from SRAM timing and control logic. The hash is
independently testable and simplifies tage_cntrl.

## Background
TAGE uses five tables (T0-T4). T0 is the base table with a
direct unhashed index. T1-T4 are tagged tables that XOR the
PC with folded global history to produce the index, and XOR
the PC with two folded history fields to produce the tag.

All folded history values arrive pre-computed from bp_history
via bp_folded_hist_t. This module does not touch history
registers directly.

The module supports one or two prediction slots via a
generate-gated NUM_PRED_SLOTS parameter. Both slots share
fld_hist_p0. Only the PC differs per slot.

## Binding Previous Decisions
- T0 index is pc[12:2] direct, no XOR. T0 is not associative.
- T1-T4 index: pc[12:2] XOR tage_tN_idx_fh. Both 11b.
  Narrower fh fields zero-extended to 11b before XOR.
- T1-T4 tag: (pc[12:2] XOR tage_tN_tag_fh1 XOR
  tage_tN_tag_fh2)[7:0]. Narrower fh fields zero-extended
  to 11b before XOR. Result truncated to TAGE_TAG_BITS=8b.
- Slot 1 uses pc_1_p0 with the same fld_hist_p0.
- Slot 1 outputs are generate-gated on NUM_PRED_SLOTS==2.
- Port naming convention: <signal>_<slot>_<pipestage>.
- Package import order: bp_defines_pkg then bp_structs_pkg,
  file-scope before module declaration.

## Specific Requirements

### Module name and file
tage_hash.sv

  NUM_PRED_SLOTS : int  -- default 1. Elaboration-time only.
                          1 or 2. Value 3+ is undefined.

### Hash computation

  No XOR. Direct PC bits only.
