### Background
The loop predictor is the second s1-stage predictor in the BP
cluster. It detects backward branches with constant iteration
counts and predicts loop exit. It overrides uBTB at s1 when
pred_is_loop is set and confidence is at maximum. It does not
participate in the s2/s3 override chain.

The design follows the Seznec TAGE-SC-L loop predictor
mechanism: each entry tracks past_itr (known loop count),
curr_itr (iterations so far this traversal), a confidence
counter, and an age counter for replacement. Prediction is
trusted only when conf == LP_CONF_LEVEL (all bits set).

No read-modify-write across the predictor. All fields needed
for update are computed at prediction time and returned in
lp_pred_t. The update path uses these stored fields directly.

     pc         : VA_WIDTH

     curs       : LP_ITR_BITS
     curs_v     : 1b
     victim     : $clog2(LP_TBL_WAYS)

4. Implement loop_pred.sv. Parameters (all overridable):
     LP_TBL_ENTRIES = 256
     LP_TBL_WAYS    = 4
     LP_TAG_BITS    = 14
     LP_ITR_BITS    = 14
     LP_CNF_BITS    = 2
     LP_AGE_BITS    = 8
     LP_N_SETS      = LP_TBL_ENTRIES / LP_TBL_WAYS
     LP_IDX_BITS    = $clog2(LP_N_SETS)
     NUM_PRED_SLOTS = 1

   Ports:
     clk, rstn
     pred_pc_p0   : input  [VA_WIDTH-1:0]
     pred_valid_p0: input
     pred_p0      : output lp_pred_t
     upd_p0       : input  lp_upd_t
     upd_valid_p0 : input

5. Index and tag derivation -- use these exact functions:

   function automatic logic [LP_IDX_BITS-1:0]
       idx_of(input logic [VA_WIDTH-1:0] pc);
     logic [VA_WIDTH-1:0] x;
     x = pc ^ (pc >> 1) ^ (pc >> 4);
     return x[LP_IDX_BITS-1:0];
   endfunction

   function automatic logic [LP_TAG_BITS-1:0]
       tag_of(input logic [VA_WIDTH-1:0] pc);
     logic [VA_WIDTH-1:0] x;
     x = pc ^ (pc >> 6) ^ (pc >> 12);
     return x[LP_TAG_BITS-1:0];
   endfunction

6. Prediction pipeline (combinational, registered at s1):
   - Compute req_idx and req_tag from pred_pc_p0.
   - Read all four ways at req_idx.
   - Find matching valid way (tag match). If hit and
     conf == LP_CONF_LEVEL: pred_is_loop=1.
   - Predict taken if curr_itr < past_itr.
   - Predict not-taken (exit) if curr_itr == past_itr.
   - If miss or conf < LP_CONF_LEVEL: pred_is_loop=0,
     pred_taken=0.
   - Compute victim using the priority policy below.
   - Output all lp_pred_t fields registered at end of s1.

7. Victim selection policy (implement as priority logic,
   do not use the exact casez structure below -- derive
   clean RTL from the policy description):
   Priority 1: lowest-indexed invalid way.
   Priority 2: if all ways valid, lowest-indexed way
               with age == 0.
   Priority 3: if all ways valid and no zero-age way,
               select the way with age less than way[0];
               check way[1] first, then way[2], then
               way[3]; default to way[0] if none qualify.

8. Update logic (synchronous, driven by upd_valid_p0):
   Use upd_p0 fields directly -- no re-read of the table.

   On taken backward branch hit (upd_p0.actual_taken &&
   upd_p0.pred_is_loop):
   - Increment curr_itr (saturate at LP_ITR_BITS max).
   - If curs_v: increment curs (saturate).

   On not-taken exit hit (upd_p0.actual_taken == 0 &&
   upd_p0.pred_is_loop):
   - If curr_itr == past_itr (correct exit prediction):
       increment conf (saturate at LP_CONF_LEVEL).
       copy curr_itr to past_itr. reset curr_itr to 0.
       reset age to maximum.
   - If curr_itr != past_itr (wrong exit prediction):
       reset conf to 0.
       copy curr_itr to past_itr. reset curr_itr to 0.

   On mispredicted exit (predicted not-taken, actually
   taken, upd_p0.pred_is_loop && !upd_p0.pred_taken &&
   upd_p0.actual_taken):
   - Reset conf to 0. Reset curr_itr to 0.

   On miss (upd_valid_p0 && !upd_p0.pred_is_loop
            && upd_p0.actual_taken
            && upd_p0.target < upd_p0.pc):
   - Allocate at victim way from upd_p0.victim.
   - Initialize: past_itr=0, curr_itr=1, conf=0,
                 age=max, v=1, curs=0, curs_v=0.
   - tag = upd_p0.tag.

9. Add sim_loop target to Makefile following the pattern
   of sim_ubtb. Include -Wno-VARHIDDEN on sim_loop only.
   Update the all target to include sim_loop.

10. Write tb/tb_loop_pred.sv with the following testcases.
    All testcases are self-checking with PASS/FAIL output.

    TC1: Miss returns pred_is_loop=0.
    TC2: Allocate on miss. Verify entry written at correct
         index and way.
    TC3: Hit with conf below LP_CONF_LEVEL returns
         pred_is_loop=0.
    TC4: Confidence builds to LP_CONF_LEVEL after correct
         exit predictions.
    TC5: Trusted prediction -- curr_itr < past_itr predicts
         taken.
    TC6: Trusted prediction -- curr_itr == past_itr predicts
         not-taken (exit).
    TC7: Correct exit increments conf, copies curr_itr to
         past_itr, resets curr_itr, resets age to max.
    TC8: Wrong exit resets conf to 0, copies curr_itr to
         past_itr, resets curr_itr.
    TC9: Mispredicted exit (predicted NT, actually T) resets
         conf and curr_itr.
    TC10: Victim selection -- all ways valid, verify
          priority policy selects expected way.
    TC11: Way conflict -- two PCs mapping to same set,
          verify independent tracking.
    TC12: curr_itr saturates at LP_ITR_BITS max, no
          overflow.
