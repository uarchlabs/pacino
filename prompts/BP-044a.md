## Assumptions made not explicit in the prompt

1. TC-CTR-R2-INC and TC-CTR-R6-DEC use IT1 as both the alternate
   provider in the update (using_primary=0, alt_comp=1) and as the
   primary provider in the readback prediction. IT2 is absent so IT1
   is promoted to primary on the subsequent prediction. The CTR
   written to alt slot is read back as prm_ctr. This is valid because
   ittage_table is index-addressed and the tag/ctr layout is shared
   between primary and alternate reads; which role a table plays is
   determined by matching comp value, not by a separate RAM bank.

2. For Phase B allocations in all new CTR tests, injecting
   prm_idx=alc_idx is required because upd_index_u0 = using_primary
   ? prm_idx : alt_idx. Without this, the allocation writes to
   IT1[0] while the subsequent prediction reads IT1[alc_idx], missing
   the entry. TC-P03 worked coincidentally because 0x3000 maps to
   IT1 idx=0x00, which equals the default prm_idx='0.

## Decisions made not explicit in the prompt

1. DIFF=0 (prm_tgt==alt_tgt) used in TC-CTR-R2-INC and
   TC-CTR-R6-DEC to suppress USE writes. Isolates CTR testing from
   USE state changes that could affect subsequent USE-dependent tests.

2. alt_ctr=3 (non-zero) used in TC-CTR-R6-DEC Phase D to suppress
   tgt_wr (tgt_wr fires when CTR reaches 0 on DEC). Keeps target
   stable for Phase E verification.

3. TC-CTR-SAT-DEC uses resolved_target=0x4000 (same as entry
   target). prm_ctr=0 + mispredict fires tgt_wr; providing the same
   value avoids replacing the tag entry which would cause Phase E
   to miss.

4. clr() called at the start of each new CTR task. Avoids any
   residual state from prior tests affecting the IT RAM for the
   fresh entry allocation.
