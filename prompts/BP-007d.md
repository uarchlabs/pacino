## Background
tage_table.sv was generated during BP-007a/b without
knowledge of two allocation ports specified in
tage_table_interfaces.md:

  alc_tbl_sel_u0  -- selector input, width TAGE_TBL_SEL_WIDTH.
                     Driven by tage_cntrl. Gating signal for
                     the allocation write to this table.
  alc_index_u0    -- index input, width THIS_IDX_BITS.
                     The RAM address for the allocation write.

tage_cntrl.sv already declares and drives these as output
ports (t_alc_tbl_sel_u0 and t_alc_index_u0). They are
currently left unconnected at the tage.sv top level because
tage_table.sv does not have the matching input ports.

Each tage_table instance knows its own table identity via
the THIS_TABLE parameter. Allocation write to a given
tage_table instance is gated on:
  alc_tbl_sel_u0 == THIS_TABLE

The allocation write writes a new entry into the DATA RAM
(and TAG RAM for T1-T4) using alc_index_u0 as the address,
with appropriate initial field values:
  - VALID = 1
  - CTR   = weakly taken (MSB = 1, remaining bits = 0)
  - USE   = 0 (useful counter reset)
  - EPC   = 0
  - TAG   = tag input on the update path (upd_tag_u0)

For T0: allocation writes only the 2b CTR with weakly taken
value (2'b10). T0 has no TAG, USE, EPC, or VALID fields.

The allocation write must be suppressed (no write issued)
when alc_tbl_sel_u0 != THIS_TABLE.

## Binding Previous Decisions
- Each tage_table instance is self-contained storage.
  tage_cntrl is the control plane; it does not instantiate
  tage_table. They are co-instantiated in tage.sv.
- THIS_TABLE parameter identifies this instance.
  Allocation gating: alc_tbl_sel_u0 == THIS_TABLE.
- Banks are per-slot RAMs. RAM0 serves slot 0, RAM1 serves
  slot 1. This applies to allocation writes as well.
  alc_tbl_sel_u0 and alc_index_u0 are slot-agnostic:
  one allocation per update event, targeting one slot.
  The slot is identified by a separate signal on the update
  path. See tage_table_interfaces.md for the full update
  port list.
- tage_alloc_comp == 0 (T0) is the no-candidate sentinel.
  When tage_cntrl drives alc_tbl_sel_u0 == 0, it means
  no allocation. T0 itself never receives an allocation
  write. Suppress all allocation writes when
  alc_tbl_sel_u0 == 0.
- Port naming convention: <signal>_<slot>_<pipestage>.
  Update path uses _u0 suffix. These two ports follow that
  convention.
- ALLOC_DATA_WIDTH = CNTRL_BITS_WIDTH + THIS_TAG_BITS.
  Allocation data is the full entry width for this table.

## Assumptions made not explicit in the prompt
A1. Allocation write takes priority over CTR/USE/EPC writes
    on the same cycle for both address mux and data mux.
    The din_mux already checked alc_we first; addr_mux now
    follows the same precedence.
    tage_table_interfaces.md does not explicitly state this
    priority; it is documented here as a design decision.

A2. Port width for alc_tbl_sel_u0 uses the module parameter
    TBL_SEL_WIDTH (=$clog2(TAGE_NUM_TABLES)+1 = 4b), not
    TAGE_TBL_SEL_WIDTH from the package (=3b). The prompt
    states TAGE_TBL_SEL_WIDTH; the interface document and
    the existing selector ports prm_tbl_sel_u0 and
    alt_tbl_sel_u0 all use TBL_SEL_WIDTH. TBL_SEL_WIDTH
    was chosen for consistency with the existing module.

A3. Port width for alc_index_u0 uses MAX_IDX_WIDTH (=11b),
    consistent with the interface document and all other
    index ports in the module. The prompt states THIS_IDX_BITS
    but the interface document shows MAX_IDX_WIDTH. The value
    is truncated to THIS_INDEX_BITS inside addr_mux as with
    upd_index_u0 and tbl_ri_wa.
