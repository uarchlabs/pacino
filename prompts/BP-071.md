## Hypothesis
bp_history.sv produces folded histories in the wrong bit order and
its incremental and recompute paths are not equivalent. Replacing
the fold with the Xiangshan FoldedHistory convention -- one fold
definition driving both update and recompute -- makes the
incremental fold and the post-rollback recompute identical for the
same history, and produces the bit order the TAGE/ITTAGE table
hashes assume.

## Background
The fold geometry in bp_history.sv was found to diverge from the
Xiangshan FoldedHistory it mimics. Three defects:

1. Insertion end. fold_step does (f << 1) | new_bit, inserting the
   newest bit at fold position 0. Xiangshan circular_shift_left
   inserts the newest bit at the HIGH end of the folded register.

2. Wrap-out position. fold_step removes the bit leaving the window
   at f[0]. Xiangshan removes it at position (len-1) % compLen
   (oldest_bit_pos_in_folded). These differ whenever
   (H-1) % W != 0.

3. Two fold definitions. The incremental path uses fold_step
   (shift-and-xor); the rollback path uses fold_ghr (a forward
   linear walk, offset i -> position i). They are not equivalent,
   so after any rollback the recomputed fold does not match what
   the incremental path had built. Xiangshan derives the recompute
   from the SAME fold definition as the update.

The wrap-out FETCH index is correct and is retained: the bit
leaving an H-deep window is read at ghr_mem[ptr + H], matching
Xiangshan oldest_bit_to_get_from_ghr -> ghr(histPtr + (i+1)).

The TAGE and ITTAGE table hashes (tage_table_hash_rules.md,
ittage_table_hash_rules.md) are CORRECT and Xiangshan-faithful:
  index: (PC >> INST_OFFSET) ^ fh
  tag:   (PC >> THIS_INDEX_BITS) ^ fh1 ^ (fh2 << 1)
They consume the folded value by XOR into the low bits and assume
the Xiangshan fold bit order. They are NOT changed by this task.
Only bp_history.sv produces the fold, so only bp_history.sv is
fixed; the hashes already expect the corrected order.

The Xiangshan FoldedHistory reference convention (taken as
authoritative for this task; do not seek external confirmation):
  - The folded register holds compLen (= W) bits.
  - Update: circular_shift_left of the folded register by
    max_update_num, newest history bit(s) XORed in at the high
    end; the bit(s) leaving the len-deep window are XORed out at
    position (len - 1 - i) % compLen for the i-th update bit; the
    leaving bit value is read from the GHR at histPtr + (i + 1) +
    (len - 1), i.e. the oldest in-window bit.
  - Recompute: re-derive the folded value from the GHR by the same
    folding definition (fold the len in-window bits into compLen by
    the same position mapping), NOT a separate forward walk.

## Defect-by-defect: what changed and where
Defect 1 -- insertion end.
  Before: fold_step did f = (f << 1) | new_bit, placing the newest
  bit at fold position 0.
  After: fold_step does a circular_shift_left by one
  (((f << 1) | (f >> (W-1))) & mask) then XORs the newest bit in at
  the HIGH end, f[W-1] ^= new_bit.
  Location: function fold_step in bp_history.sv.

Defect 2 -- wrap-out position.
  Before: the leaving bit was removed at position 0
  (f[0] = f[0] ^ ... ^ bit_out).
  After: the leaving bit is removed at wrap_pos = (H - 1) % W,
  f[wrap_pos] ^= bit_out. This requires the window depth H, so
  fold_step gained an H argument; every call site passes the
  table's *_TBL_HIST value (the same constant already used to fetch
  bit_out at ghr_mem[ptr + H]). The leaving-bit FETCH index
  (ptr + H slot 0, ptr + 1 + H slot 1) is unchanged, per the
  prompt.
  Location: function fold_step plus all fold_step call sites.

Defect 3 -- single-definition recompute.
  Before: rollback used fold_ghr with a forward walk pos = i % W,
  inequivalent to the incremental fold_step.
  After: fold_ghr uses the SAME position mapping the incremental
  step builds, pos = (i + W - 1) % W (newest at W-1, one left
  rotate per older bit, leaving bit at (H-1) % W = posmap(H)).
  Algebra: step(fold_ghr(mem, p+1, H, W), mem[p], mem[p+H], H, W)
  == fold_ghr(mem, p, H, W). Verified offline over the in-use
  (H, W) set incl. H == W (SC) and H > W (TAGE/ITTAGE).
  Location: function fold_ghr (pos formula); rollback recompute
