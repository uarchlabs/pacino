## Hypothesis

A self-contained bp_history module can maintain GHR and PHR as
circular buffers with externally-driven pointers, compute all
folded histories incrementally, recompute folds on rollback,
and pass directed Verilator tests covering update, rollback,
and fold correctness.

## Background

The history module is the centralized source of branch history
state for the BP cluster. It owns:
  - GHR: 256b circular buffer, pointer ghist_ptr (8b)
  - PHR: 32b circular buffer, pointer phist_ptr (5b)
  - All folded histories for TAGE (T1-T4), ITTAGE (IT1-IT4),
    and SC (ST1-ST3) for index and tag computation

The circular buffer pointers are driven externally. The buffer
storage lives inside this module. On rollback, the external
logic supplies a new pointer value -- no decrement arithmetic
inside this module.

Folded histories are maintained incrementally during normal
operation. On rollback they are recomputed from the circular
buffer contents at the restored pointer position. This is a
known timing concern deferred to a future session (G15 in
bp_cluster.md).

PHR/GHR mixing for index and tag hashing in TAGE and ITTAGE
is TBD -- resolved at TAGE/ITTAGE implementation time. This
module exposes both raw registers and all folded histories.

All predictor table sizes, history lengths, and fold widths
are specified in bp_cluster.md. Read that file for the
authoritative values -- do not use training data.

The module supports dual prediction (NUM_PRED_SLOTS=1 or 2).
Up to two update bits may be written per cycle.

## Binding Previous Decisions

## Assumptions made
- num_branches value 2'b11 is unused; only 0, 1, 2 are valid.
- Rollback does NOT clear ghr_mem or phr_mem. Only the pointer
  context is restored. Folds are recomputed from the current
  buffer state at the restored pointer. TC8 reference model
  reflects this: the reference fold is computed from the
  post-advance ghr_buf, not the pre-advance snapshot.
  This matches the spec ("GHR and PHR memory contents are NOT
  cleared") and is architecturally sound for this pointer-based
  circular-buffer design.
- ckpt_ghist_ptr and ckpt_phist_ptr track the most recently
  written checkpoint pointer values as combinational outputs
  (latched on ckpt_wr_en). They do not change on rollback.
- The loop variable idx_i in the reset clause is declared at
  module scope (integer) to avoid Verilator limitations on
  local variable declarations inside always_ff blocks.
