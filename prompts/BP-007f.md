## Assumptions made not explicit in the prompt
- Tag and index PC encoding for the TB: the prompt required
  hash-derived test vectors but did not specify how to derive
  consistent (idx, tag) pairs from a single PC. Chose tags with
  bits[1:0]==00 and indices with bits[10:9]==00 to avoid
  conflicting bit assignments at pc[12:11]. With fh=0 this
  gives idx_hash=pc[12:2] and tag_hash=pc[18:11] independently.
  This is a TB implementation detail, not an RTL change.
- For TC10 (dual-slot simultaneous predict), each slot encodes
  its own tag via its own PC value. Slot 0 uses PC_TC9_S0
  (tag=0xAC), slot 1 uses PC_TC9_S1 (tag=0xBC). folded_hist
  is shared at 0 for both.
- INST_OFFSET=2 is defined as a module localparam. It is not
  currently in bp_defines_pkg.sv. This constant encodes the
  RISC-V 4-byte instruction alignment offset. If it is later
  added to bp_defines_pkg.sv, the localparam should be removed.

## Decisions made not explicit in the prompt
- idx_hash and tag_hash are declared at module scope as
  logic arrays (not inside always_comb) to avoid Verilator
  concerns with variable declarations in procedural blocks.
- tag_hash is declared MAX_TAG_WIDTH wide (same as tag_hash_p1)
  with inner THIS_TAG_BITS truncation and outer MAX_TAG_WIDTH
  zero-extension. This avoids WIDTHEXPAND/WIDTHTRUNC warnings.
- fh_idx_ext, fh1_ext, fh2_ext are declared VA_WIDTH wide to
  allow the XOR with the shifted PC in a single expression
  without intermediate width mismatch.
- The fh_sel always_comb default case assigns 0 to all three
  extended fields. This handles both T0 (which does not use
  folded_hist) and any out-of-range THIS_TABLE value.
- In the TB the tbl_ri_wd for TC8 was changed from 16'hABCD
  to 16'hACCD (tag byte 0xAB -> 0xAC) so the RI-written entry
  can be predicted using the same PC formula as all other tests.
