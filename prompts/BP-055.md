## Hypothesis

The five ITTAGE write paths (CTR, USE, allocation, EPC, TGT),
each proven in isolation, operate correctly when exercised
together in one directed sequence with no cross-path
interference. In particular, an allocation writes only the
selected table's RAM and leaves non-selected tables unchanged
at the RAM level.

## Background

#72 is the ITTAGE unit capstone. Its gate -- #56 (EPC), #57
(TGT), #59 (UAON), #61 (aging), #63 (allocation), #65
(prediction-side) -- is closed, each proven alone. The capstone
is run only now, after isolation, for the same reason as #71:
mixing paths before each is individually proven reproduces the
multi-cause ambiguity that stalled BP-044.

BP-053 proved the controller emits the correct per-table
allocation selector but did not prove RAM-level write isolation
(TC-ALC-11 ran at the controller, with no table instantiated).
At sim_ittage the tables are present, so the capstone closes
that seam: an allocation step reads back the selected table
changed and a non-selected table unchanged.

## Binding Previous Decisions

- EPC write condition is a strict subset of the CTR write
  condition: no path writes EPC without also writing CTR
  (BP-050b). When checking EPC landing, drive a CTR-writing
  update.
- Allocation index is pre-hashed at its source; seed the
  allocation target index accordingly (BP-053).
- UAON counter movement requires a two-hit transaction; a
  single-hit transaction holds the counter by the single-hit
  guard (BP-051). Where the sequence intends UAON movement,
  drive a two-hit setup.

## Other Notes

The EPC+USE interaction: when EPC != lcl_epoch at update
time, u_eff < raw USE. Writing u_eff+1 to the USE field
may leave raw USE unchanged (as in Phase 2: u_eff=0,
write=1=seed). The EPC write simultaneously resets age to
0. This is correct architectural behavior: the raw USE
field tracks what u_eff+1 was at the time of the last
update, and EPC records when that update happened.
Allocation candidate selection uses u_eff (not raw USE),
so the correct protection semantics are preserved.
