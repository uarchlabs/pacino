## Assumptions made not explicit in the prompt

- TAG field in contamination entry (u_ram_s0) for slot1_pred_tst
  uses same TAG value (0x0A) as the correct u_ram_s1 entry.
  This ensures T1 would still "hit" if contamination occurred,
  making the CTR value (100 vs 111) the distinguishing signal.
  An alternative (different TAG) would cause a miss and fall
  back to T0 -- also detectable but less direct.
- prm_tkn=1 used in slot1_upd_tst meta struct for CTR=011
  (weakest not-taken), consistent with upd_prm_inc_tst convention.
  The update logic uses prm_tkn vs resolved_taken to determine
  INC/DEC, not CTR MSB directly.

## Decisions made not explicit in the prompt

- T1 chosen as the tagged table for both tests (simplest to
  reason about; uses the same formula as T0 for idx with
  folded_hist=0 but adds tag comparison).
- PC=40'h5000 chosen for slot1_pred_tst: produces idx=11'h400
  (bank=1 row=0), different from all existing slot 0 test
  addresses which use bank=0 rows.
- prm_idx=11'h046 (row=70) chosen for slot1_upd_tst: distinct
  from all existing slot 0 update test addresses.
- Slot 1 staging variables placed adjacent to slot 0 staging
  variables, extending the same always_ff blocks as required.
