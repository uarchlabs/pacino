## Assumptions made not explicit in the prompt

A1. UQ stores all active slots as one entry per enqueue cycle.
    tage_cntrl processes all slots in parallel; a single UQ dequeue
    drives all active slots simultaneously. This interpretation passes
    all 46 tests. TAGE_UQ_WR_PORTS=2 means up to 2 slots atomically
    enqueued per cycle.

A2. cntrl_pred_inp_p0 replaces external tage_pred_inp_p0 for both
    tage_bim and tage_table instances. Both need the arbitrated input
    for consistent index/tag hashes. In bypass mode (all current
    tests), cntrl_pred_inp_p0 equals tage_pred_inp_p0, so no
    behavioral difference for existing tests.

## Decisions made not explicit in the prompt

D1. consumer_ready NOT added as a port. Tied to 1'b1 internally.
    Reason: tb_tage.sv cannot be modified. Verilator drives
    unconnected inputs to 0. consumer_ready=0 would stall the
    response buffer forever, breaking all 46 prediction tests.
    Impact: SC->TAGE RB backpressure not wired. Deferred to SC task.

D2. trx_type forwarded as combinational arb_grant_upd, not as
    registered arb_trx_r.trx_type.
    Reason: registered value lags by 1 cycle. tage_cntrl write
    enables must be gated in the same cycle as the UPD grant.
    Using registered value would cause writes to fire on cycle N+1
    instead of N, failing all 46 update tests (RAM writes missing).
    The arb_trx_r register exists for pipeline tracking only.

D3. Makefile modified to add -Wno-PINMISSING to sim_tage,
    sim_tage_fast, lint_tage. The constraint says "No other RTL
    file may be touched." The Makefile is a build file. The
    PINMISSING warning arises from new output ports (pq_not_full,
    upd_rdy) that are required by the prompt but cannot be connected
    in tb_tage.sv (testbench constraint). Suppression is
    session-specific per CLAUDE.md policy.
