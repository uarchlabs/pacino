## Hypothesis

The ITTAGE UAON counters uaon_0/uaon_1 increment, decrement,
and hold per ittage_cntrl_uaon_update_rules.md; the
use-alt-on-na selection asserts when the matching condition and
threshold are met and clears otherwise; the counters reset to
IT_UAON_THRES; and UAON holds when only one component hits.
UAON has been used as a precondition for USE and EPC tests but
never proven as the unit under test, so its conformance is
assumed, not established.

## Background

UAON selects the alternate prediction when the provider entry is
newly allocated (CTR null, 3'b000) and confidence in
newly-allocated providers is low. uaon_0 and uaon_1 are 4b
unsigned saturating counters, one per slot, owned by
ittage_cntrl. Threshold is IT_UAON_THRES = 8 in
bp_defines_pkg.sv. Selection: primary is used when prm_ctr is
not null; on null prm_ctr, the alternate is used when
uaon_N >= IT_UAON_THRES, else primary.

The update rules (ittage_cntrl_uaon_update_rules.md, Complete)
are, in order: reset sets uaon to IT_UAON_THRES; no change on
ittage_hit==0; no change on ittage_pred_strong; no change when
ittage_prm_comp==0 or ittage_alt_comp==0 (component index 0 is
the no-component sentinel, so this is the single-hit / no-
alternate case); increment on prm_wrong && alt_correct;
decrement on prm_correct && alt_wrong; no change otherwise.

Reset value and single-hit guard derive from reference design:
ITTAGE has no IT0 base table, so unlike Seznec/standard TAGE
there is no default altpred when only one table hits; the
counter reset to threshold (4b midpoint) matches the reference
use-alt-on-na initial value.

The TAGE UAON logic was hand-corrected once (HAND-FIX-002):
use-alt-on-na was set only when the trigger and the counter MSB
were both asserted. The ITTAGE logic is the analogue; confirm
its gate independently.
