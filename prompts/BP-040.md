ittage_cntrl.sv (three bugs exposed after trx_type_tb=0 wired):

  Bug B: trx_type && upd_val gating on all four update write blocks.
    In the full ittage system, the arbiter ensures upd_val is non-zero
    only when trx_type=1, so the gate is redundant in that context.
    In standalone TB with trx_type_tb=0 hardwired, the gate blocks
    every update write. Removed trx_type && from all four guards;
    update writes now fire on upd_val alone, matching the full-system
    invariant.
    Affected blocks: CTR (alt_ctr_wr, prm_ctr_wr), USE/EPC,
    TGT, ALC.

  Bug C: prm and alt CTR write signals swapped in g_ctr_upd block.
    if (using_primary) branch was writing to prm_ctr_wr_u0 and
    prm_ctr_wd_u0 from ittage_prm_ctr. TC-UPD-01 and block header
    comment both specify: when using_primary=1, write alt_ctr_wr_u0
    (saturating alt toward taken) not prm_ctr_wr_u0. The else branch
    had the mirror error. Swapped both branches to match spec.

  Bug D: alc_index_u0 assigned from ittage_alc_idx field of
    ittage_pred_meta instead of upd_index_u0[s].
    TC-UPD-06 expects alc_index_u0[0] = 9'h033 = prm_idx, which
    equals upd_index_u0[s]. ittage_alc_idx in the meta is 9'h044
    (a different pre-allocated candidate index). Changed assignment:
    alc_index_u0[s] = upd_index_u0[s];

ittage.sv (TD #50 FAST_INIT fix):
  Inside gen_ittage_tables generate loop, tbl_ri_active was passed
  directly as ri_active to each ittage_table port. Per sram_init.md,
  all sram_init control signals must be gated to suppressed values
  when fast_init is active. Added:
    logic tbl_ri_active_w;
    assign tbl_ri_active_w = fast_init ? 1'b0 : ri_active;
