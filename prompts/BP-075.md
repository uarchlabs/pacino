## Hypothesis
The SC ST0-ST3 table is a counter-only RAM table with no tag, no
useful field, no valid bit, and no allocation path. It can be
implemented as a parameterized module wrapping two bw_ram instances
(one per slot), an index hash on the prediction path, a direct
counter read output, and a single counter write on the update path.

## Background
sc_table implements SC tables ST0-ST3. Each SC entry is a single
signed confidence counter of width SC_TBL_CTR. SC entries have no
tag, no USE field, no EPC field, and no valid bit. There is no
allocation. The prediction path reads the counter; the update path
overwrites it with the saturated stepped value supplied by sc_cntrl.

Port list and semantics: sc_table_interfaces.md.
Index hash: sc_table_hash_rules.md (sc_idx_hash).
Field widths: bp_defines_pkg.sv (SC parameters).

tage_table.sv is the structural pattern reference for the two-RAM
per-slot layout, the bw_ram bank decomposition, and the tbl_ri
override path. sc_table is simpler: no tag compare, no useful field,
no EPC, no valid bit, no allocation.

## Binding Previous Decisions
- Two module types: sc_table (ST0-ST3), sc_brimli (ST4). This task
  is sc_table only.
- Each table contains two bw_ram instances, one per prediction slot.
  bw_ram instantiated with BANKS=2 (sc_decisions.md section 6).
- Bank decomposition: bank_addr = index[THIS_INDEX_BITS-1],
  row_addr = index[THIS_INDEX_BITS-2:0]. Identical to tage_table.
- No allocation path. Update is a single counter write per slot.
- Index hash is prediction-path only. The update path takes the
  precomputed index on upd_index_u0.
- Prediction index applied at p2. Counter output ctr_p3 at p3
  (bw_ram one-cycle read latency).
- ST0 uses fh_idx_ext=0 (SC_TBL_FH[0]=0, unhashed PC slice). The
  idx_fh_p2 port is tied to constant zero at the ST0 instance in
  sc.sv; port presence is not parameterized away.

## Specific Requirements

   tage_table.sv before writing.
2. Module name sc_table, file sc_table.sv. One module per file.

   from the SC parameters in bp_defines_pkg.sv. Do not hardcode
   table-specific values.
4. Instantiate two bw_ram (BANKS=2), one per prediction slot, per
   the port list in sc_table_interfaces.md. Data width is the SC
   counter width. No tag, no useful, no EPC, no valid bit.
5. Prediction path: compute the index per sc_idx_hash
   (sc_table_hash_rules.md):

                    ^ fh_idx_ext)
   fh_idx_ext is the idx_fh_p2 input (the instance-selected fold;
   ST0 instance ties it to zero). Expose idx_hash_p2[s] as an

   Present ctr_p3[s] as the counter read output.
6. Update path: write the counter at upd_index_u0[s] when
   ctr_wr_u0[s] is asserted, data ctr_wd_u0[s]. Use the same bank
   decomposition. No read-modify-write.

   tbl_ri_wr / tbl_ri_wa / tbl_ri_wd override all other write
   controls, addresses, and data to the bw_ram instances. Same
   scheme as tage_table.
8. Prediction and update paths are mutually exclusive by phase. Do
   not implement read-modify-write or read-during-write forwarding.
9. Add an assert on the sc_idx_hash THIS_TABLE case default arm

## Assumptions made not explicit in the prompt
- ALLOC_DATA_WIDTH is derived from the SC_MAX_*_WIDTH package values
  (CNTRL_BITS_WIDTH = VAL+CTR+USE+EPC, plus TAG), which reduce to the
  counter width (6b) since all non-CTR SC widths are 0. This matches
  the interface Derived Parameters (ALLOC_DATA_WIDTH = SC_TBL_CTR).
- The counter write enable is gated by sc_upd_val_u0[s] & ctr_wr_u0[s].
  The interface states sc_upd_val_u0 enables the write and ctr_wr_u0
  is the counter strobe; both are required. This mirrors tage_table's
  tage_upd_val_u0 & prm_ctr_wr_u0 gating.
- sc_pred_val_p2 is retained as a port for interface completeness but
  is not wired: bw_ram has no read enable and reads are harmless.
  Covered by the project-wide -Wno-UNUSED.
- Instances named u_ram_s0 / u_ram_s1 to match the fast-init
  hierarchical-reference pattern documented in sc_decisions.md
  section 13.

## Decisions made not explicit in the prompt
- bweb_n is tied to all-zero (all bits writable) for both slots. The
  SC entry is a single whole-word counter with no field
  decomposition, so no partial bit-write is needed; wen_n alone gates
  the commit.
- The PC slice is widened to SC_MAX_FH before the shift/XOR
  (SC_MAX_FH'(inp_pc_p2[s])) so both XOR operands share the same
  bit-width context. Without this Verilator raises WIDTHEXPAND
  because the XOR forces a 64-bit context onto the 39-bit PC slice.
  The final result is truncated to THIS_INDEX_BITS by the cast, which
  matches the specified hash.
- The requirement-9 assert is implemented as an immediate assertion
  (assert (THIS_TABLE <= 3) else $error(...)) inside the fold-select
  case default arm. It is unreachable by construction for a valid
  instantiation (THIS_TABLE 0-3) and lints clean.
