## Hypothesis

The four TAGE write paths (CTR, USE, allocation, EPC), each
proven in isolation, operate correctly when exercised together
in one directed sequence on overlapping entries, with no
cross-path interference: each field lands as the path rules
specify, no path corrupts a field written by an earlier step,
and an allocation does not disturb entries written by prior
updates.

## Background

#71 is the TAGE unit capstone. Its gate -- #55 (EPC), #58
(UAON), #60 (aging), #62 (allocation), #64 (prediction-side) --
is closed, each proven alone. The capstone is run only now,
after isolation, for the same reason as the ITTAGE capstone
(#72): mixing paths before each is individually proven
reproduces the multi-cause ambiguity that stalled BP-044.

The ITTAGE capstone (BP-055) hit a non-discriminating USE check
when a seeded EPC differed from the epoch (age != 0 made the USE
increment write u_eff+1, which equaled the seed). This flow
avoids that: aging is held disabled and EPC is seeded equal to
the epoch (age 0), so u_eff equals raw USE and every USE delta
is visible. EPC writes still occur on update (EPC rides the
USE/CTR write per #55), so EPC landing is still exercised; the
aging interaction itself is #60's scope, not the capstone's.

## Binding Previous Decisions

- TAGE has no target field. The flow is CTR, USE, allocation,
  EPC -- there is no TGT step (this is where the flow differs
  from BP-055).
- EPC write rides the USE/CTR write (HAND-FIX-001, #55):
  EPC = lcl_epoch is written in the same transaction as the
  CTR/USE update.
- Aging held disabled; EPC seeded equal to the epoch so age = 0
  and USE deltas are visible.
- Allocation RAM-level write isolation is already proven
  (#62, BP-059 TC-95). This flow tests allocation against prior
  CTR/USE/EPC writes; do not re-run single-path isolation.
- Allocation targets T1-T4; T0 is never an allocation target.
- UAON forced as a read input (#58); not trained.
- Seed entries via the bw_write backdoor.

## Specific Requirements

Read the cited update-rule rows before encoding each step's
expected result. Place and hit seeded entries using the
existing seed-and-predict pattern in tb_tage.sv; decode readback
fields per tage_table_entry_formats.md. The flow is one
self-contained sequence: seed every entry it reads or writes,
carry no state across steps beyond what the sequence produces,
and force UAON and aging inputs rather than relying on defaults.

1. Seed and lookup. Invalidate aliasing entries. Seed a provider
   (a longer table, e.g. T2) and an alternate (a shorter table,
   e.g. T1) at the chosen PC, each with distinct valid, tag,
   CTR, USE, and EPC = epoch. Seed one entry in a non-target
   table to a known non-reset value as a cross-path interference
   reference. Drive the lookup and confirm the seeded provider
   is selected (read context for the update steps).

2. CTR and EPC step. Drive a CTR-writing update on the provider.
   Confirm the CTR field moves per
   tage_cntrl_ctr_update_rules.md and the EPC field is written
   to the epoch in the same transaction. Read both back and
   compare against the seeds.

3. USE step. Drive an update that changes USE per
   tage_cntrl_use_update_rules.md. With age = 0, the USE delta
   is visible (u_eff = USE). Read USE back, and confirm the CTR
   and EPC written in step 2 are intact (no cross-path
   corruption).

4. Allocation step. Drive a mispredict meeting the allocation
   trigger so an entry is allocated into a longer table. Confirm
   the allocated entry contents per the alloc-rules init, and
   confirm the provider entry written in steps 2-3 and the
   step-1 interference reference are both unchanged.

5. Final readback. Decode every touched entry (valid, tag, EPC,
   USE, CTR) per tage_table_entry_formats.md. Confirm each field
   holds the value the sequence should have produced and that no
   unrelated field moved.

## Decisions made not explicit in the prompt

- Provider table chosen as T2 (not T1) so that T1 can serve as
  a shorter alternate, making UAON meaningful: T2 is the "long"
  boundary case, T1 is the "short" correct alternate.
- T2 seeded with CTR=010 (weakest not-taken) to be 1 INC away
  from the UAON boundary (011), enabling step 3 without
  requiring extra update steps.
- T4 seeded with a valid but non-matching TAG (0x12 vs 0x45) to
  act as interference reference without affecting prediction.
- EPC seeded to 0 for all T1-T4 entries; lcl_epoch forced to 0.
  Age=0 ensures u_eff=USE (aging disabled) and EPC writes are
  no-ops (write 0 back to 0), keeping EPC verification simple.
