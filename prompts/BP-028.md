exercised. Any defect in those mux arms would be invisible
to the current test suite.

the folded history struct. It fires on the prediction path,
not the update path. To cover T2 and T3 arms, tests must
issue predictions that produce a hit in T2 and T3 instances
respectively.

## Other Notes

The fh_sel coverage gap was structural: for a generate loop of N
tage_table instances, each case arm is only reachable in its own
instance. verilator_coverage --annotate merges per-instance case
arm coverage into a single per-source-line view using the count
from the last-indexed instance. This is correct for the last
instance's own arm (T4 shows correctly) but incorrect for lower
instances (T1/T2/T3 show 0 regardless of execution). This is a
known limitation when using --coverage-line on parameterized
generate loops in Verilator.
