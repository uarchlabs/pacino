## Hypothesis

The TAGE UAON counter (uaon[s]) increments, decrements, and
holds per each row of tage_cntrl_uaon_update_rules.md, and
use_alt_on_na asserts only when uaon_trig and the counter MSB
(uaon[s][3]) are both set (HAND-FIX-002). A single-hit
transaction (only the provider matches, or only the alternate
matches) holds the counter; if TAGE lacks the guard that
enforces this, the counter moves incorrectly, which is the
defect class found and fixed in ITTAGE (#59, BP-051).

## Background

#58 is the TAGE half of the UAON proof. The ITTAGE half (#59,
BP-051) found a real RTL defect: the UAON update block was
missing the single-hit guard, so a single-hit transaction with
a stale alternate could move the counter. The fix added the
guard and was proven by removing it (the check failed) and
restoring. This task applies the same method to TAGE and
specifically tests whether the TAGE counter holds on a
single-hit transaction.

tage_cntrl_uaon_update_rules.md is Draft. This task is the
directed proof of each row; promotion of the document to
Complete is a separate post-session action. If the Draft
document and the RTL disagree, stop and report the
disagreement rather than silently matching either one
(reference BUG-001: a planning-document error once produced a
false fail).

HAND-FIX-002 gates the use_alt_on_na output
(use_alt_on_na = uaon_trig & uaon[s][3]). That is a gate on
the output, not on the counter update. A single-hit guard, if
present, would sit on the counter update and is a separate
mechanism; HAND-FIX-002 does not by itself prevent a
single-hit counter move.
