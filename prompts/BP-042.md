tage_assert.sv is bound to tage.sv via a SystemVerilog
bind statement in tb_tage_manual.sv. It checks three
invariants at the tage module boundary on every clock:
  - prm_comp=0 with alt_comp>0 is invalid (row 18)
  - tage_using_primary=1 when pCMP=aCMP=0 (ADR-001)

ittage_assert.sv is a new module following the same
pattern, bound to ittage.sv. It checks three invariants:
  1. ittage_hit=1 with pCMP=0 and aCMP=0 is impossible
  2. ittage_using_primary=1 with pCMP=0 is impossible
  3. ittage_using_primary=0 with hit=1 and aCMP=0 is
     impossible

DEFERRED-1: ittage_assert.sv multi-line string syntax.
  The content of tb/ittage_assert.sv (and rtl/ittage_assert.sv
  written from it) uses implicit string concatenation in
  $error() arguments, e.g.:

  concatenation. Compilation of sim_ittage fails with 6
  syntax errors at lines 39, 50, 67, 76, 94, 104. The
  "do not modify ittage_assert.sv content" constraint prevents
  fixing this in the current session. Fix requires merging
  the two string literals per $error() call into one string.
  This is the root cause of sim_ittage compile failure.

  present, or tage_assert.sv must be extended to gate the
  update check with an additional condition. Neither is
