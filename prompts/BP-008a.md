## Hypothesis
tage_cntrl.sv prediction-side logic can be implemented

## Background
tage_cntrl.sv is the control module for the TAGE branch
predictor. It sits between tage_table instances and the
BP cluster. It drives all table inputs, collects table
outputs, applies the provider selection policy, applies
the UAON mux, and produces the final tage_pred_meta_t
output.

This prompt covers prediction-side logic only. Update
logic is deferred to BP-008b.

The TAGE predictor has five tables. T0 is the base table:
2b saturating counter, no tag, no valid, no useful, always
treated as hit. T1-T4 are tagged tables with 3b CTR,
valid, tag, useful, and EPC fields.

NUM_PRED_SLOTS=2 throughout. Both slot 0 and slot 1 logic
are always present unconditionally.

## Binding Previous Decisions
- T0 is always-hit. It is never the alternate provider.
  tage_cntrl treats T0 as unconditionally hitting.
- Provider selection: longest-history tagged table with
  hit_p1 asserted. If no tagged table hits, T0 is
  provider. tage_prm_comp == 0 indicates T0 as provider.
- Alternate provider: longest-history tagged table with
  hit_p1 asserted that is shorter than the primary
  provider. If none, T0 is alternate.
- UAON trigger: provider CTR == 3'b011 or 3'b100
  (boundary states). When triggered and uaon counter
  >= TAGE_UAON_THRES, use alternate prediction instead
  of primary.
- tage_pred_strong reflects NOT WEAK on the final
  provider CTR after UAON mux selection. Not strictly
  the primary CTR.
- Two 4b saturating UAON counters: uaon_0 and uaon_1,
  one per slot. Both owned by tage_cntrl.
- tage_pred_rdy_0/1_p2 asserted when tage_pred_val_0/1
  _p0 was asserted two cycles prior.
- CTR direction: CTR[2] is MSB for T1-T4. CTR[1] is
  MSB for T0 (2b counter).
- Port naming: <signal>_<pipestage>. No slot in signal
  name -- slot dimension is array index.
- Banks are per-slot RAMs. RAM0 serves slot 0, RAM1
  serves slot 1. Selection is structural.
- tage_alloc_comp == 0 is the no-candidate sentinel.
  Allocation logic is present in this module but
  allocation writes are suppressed on the prediction
  side -- alloc write signals are driven to zero here
  and activated in BP-008b.
