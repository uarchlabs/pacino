### Provider role for rows 13b and 13c

Rows 13b and 13c apply when both providers are T0:
  prm_comp == 0 and alt_comp == 0.
This means no tagged table (T1-T4) hits for the test PC.
T0 is the sole prediction source. tage_pred_tkn comes
from T0 CTR MSB (bit 1 of the 2b counter).
tage_using_primary=1 (T0 is always the primary fallback).
UAON does not apply when prm_comp==0.

### T0 CTR encoding

T0 uses a 2b saturating counter. Direction is CTR[1]
(the MSB). Weak states are 01 and 10.
  00  strongly not-taken
  01  weakly not-taken
  10  weakly taken
  11  strongly taken

T0 initializes to 10 (weakest taken) under FAST_INIT.
tage_prm_ctr stores the T0 2b CTR zero-padded to 3b.
tage_prm_ctr[2] is always zero for T0 and must not be
used as the direction signal. tage_prm_pred_tkn is the
authoritative direction field, sourced from T0 CTR[1]
via t_taken_p1[0][s].

### Round-trip test pattern

Same as BP-011, BP-014a, BP-014b. Summarized:
  1. Pre-load BIM RAM entry via hierarchical reference.
  2. Predict via staging always_ff. Wait two posedge
     clk edges.
  3. Capture tage_pred_meta_p2[0].
  4. Construct tage_upd_inp_u0[0] from captured meta
     plus resolved_taken and cond_mispredict. Drive
     via staging always_ff. Wait for
     tage_upd_rdy_u1[0].
  5. Re-predict at same PC.
  6. Verify BIM RAM state and re-prediction output.

Staging always_ff is mandatory. Extend existing blocks.
Do not create new always_ff blocks.
All tests use +TAGE_FAST_INIT=1.

### T0 (BIM) RAM structure

T0 is implemented in tage_bim.sv. It contains two bw_ram
instances, one per prediction slot (RAM0 for slot 0,
RAM1 for slot 1). Each entry is a 2b CTR. No valid bit,
no tag, no useful bit, no EPC.

T0 index is derived directly from PC bits, not hashed:
  index = pc[12:2]  (11 bits, TAGE_T0_ENTRIES=2048)

Bank and row assignment:
  bank_addr = index[10]
  row_addr  = index[9:0]

BIM RAM hierarchical reference for slot 0:
  tb.u_dut.u_tage_bim.u_ram0.mem[bank][row]

Entry format: [1:0] CTR only.
  mem[bank][row] = 2'bXX  where XX is CTR[1:0].

TAGE_SRAM_INIT_VALUE=0. An entry with VALID=0 never

the test PC. The FAST_INIT zero state ensures no hits.

### Allocation at update time

cond_mispredict=1 for both tests. Allocation rules apply:
  prm_comp == 0 satisfies prm_comp < M (M=4, index of T4).
  Allocation will attempt to find a candidate in T1-T4.
  Under FAST_INIT, all T1-T4 entries have USE=00 and
  VALID=0. u_eff=00 qualifies as an allocation candidate.
  Allocation WILL fire unless suppressed.

To suppress allocation and isolate the T0 CTR update:
  Pre-load T1, T2, T3, and T4 at the test PC index with
  USE=01 (ueff=01 != 00, not a candidate).
  Pre-load value: 16'h0010 (VALID=0, USE=01, TAG=0).
  VALID=0 ensures no tag hit during prediction.
  USE=01 ensures ueff != 00 and blocks allocation.

T1-T4 index for the test PC uses the same idx_hash
formula as BP-014a/b (folded_hist=0):
