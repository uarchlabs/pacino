## Hypothesis
ittage_cntrl.sv prediction path can be implemented by

## Background
ittage_cntrl.sv is the controller for the ITTAGE indirect
target predictor. It is a pure control module: no SRAM,
no bw_ram instantiation, no ittage_table instantiation.
It receives per-table outputs from five ittage_table
instances (IT1-IT5) and drives update signals back.
sram_init and bw_ram live in ittage.sv and
ittage_table.sv respectively.

This task covers the prediction path only.

Prediction pipeline stages:
  p0: idx_hash_p0 and tag_hash_p0 arrive from each
      ittage_table instance (combinational).
  p1: hit_p1 and cntrl_bits_p1 arrive from each
      ittage_table instance. Provider scan, alternate
      provider scan, allocation candidate scan, u_eff
      computation, UAON mux decision, and
      ittage_pred_meta_t population all execute
      combinationally at p1.
  p2: ittage_pred_meta_t and ittage_pred_rdy_p2
      flopped out.

State registers owned by ittage_cntrl (NUM_PRED_SLOTS):
  uaon[0:1]                4b saturating counters
  lcl_epoch[0:1]           2b wrapping counters
  lcl_aging_interval[0:1]  32b interval counters

In this task: all state registers declared and
reset-initialized only. No update logic for any state
register until BP-034b.

## Binding Previous Decisions

1. tage_cntrl.sv is a structural reference only.
   Read it for port organization and always_comb
   coding patterns.
   Do NOT inherit:
   - T0 base table handling. ITTAGE has no IT0.
   - Direction CTR mux. CTR is confidence, not direction.
   - TAGE allocation logic. ITTAGE has the no-consecutive-
     table constraint; TAGE does not.
   All type names change: tage_* -> ittage_*,
   TAGE_* parameters -> IT_* parameters.

2. cntrl_bits_p1 field layout received from ittage_table.
   These bit positions are fixed. Do not recompute.
     Bit  0      : VAL (1b)
     Bits [3:1]  : CTR (IT_MAX_CTR_WIDTH = 3b)
     Bits [5:4]  : USE (IT_MAX_USE_WIDTH = 2b)
     Bits [7:6]  : EPC (IT_MAX_EPC_WIDTH = 2b)
     Bits [45:8] : TGT (IT_MAX_TGT_WIDTH = 38b)
   Define localparam CNTRL_BITS_WIDTH = 46.

3. Port sizing for update path output arrays.
   alc_wd_u0 must accommodate the widest allocation entry
   across all tables. Tag width varies per table (8, 8,
   9, 9, 11 bits for IT1-IT5). Use the maximum:
     ALLOC_DATA_WIDTH_MAX =
       CNTRL_BITS_WIDTH + IT_MAX_TAG_WIDTH  (= 46+11 = 57)
   Define this as a localparam.

4. Table input and output port arrays are indexed
   [0:IT_NUM_TABLES-1]. Index 0 is unused (IT0
   placeholder), consistent with the bp_defines_pkg.sv
   parameter array convention. IT1-IT5 map to indices
   1-5.

5. ittage_pred_strong definition.
   ittage_cntrl_uaon_update_rules.md defines NOT NULL:
     ctr != 3'b000
   ittage_pred_strong reflects NOT NULL on the final
   provider CTR after the UAON mux selects primary or
   alternate:
     ittage_pred_strong = (final_provider_ctr != 3'b000)
   Note: ittage_cntrl_decisions.md uses "strong states"
   to describe CTR saturation endpoints (000, 111). This
   is saturation terminology only and does not define the
   ittage_pred_strong signal.

6. All update path output ports driven to zero in this

     prm_ctr_wd_u0, alt_ctr_wd_u0, use_wd_u0, epc_wd_u0

  Two named always_comb blocks per Binding Decision 7.
  Each block covers the following for its slot index s:

     from tbl_cntrl_bits_p1[t][s] using the LSB/MSB
     localparams from Step 4.

     For t = 1 to 5, compute u_eff using the formula
     from ittage_cntrl_use_update_rules.md:

       if age == 0: u_eff = use[t]
       if age == 1: u_eff = use[t] >> 1
       else:        u_eff = 0
     u_eff is a 2b saturating value.

     Scan t = 5 down to 1. First table with
     tbl_hit_p1[t][s] == 1 and val[t] == 1 is the

     prm_tgt (from tbl_pred_tgt_p1[t][s]).
     If no table hits: set ittage_hit = 0.
     ittage_pred_rdy_p2 still asserts on no-hit.

  d. Alternate provider scan.
     Continue scan below prm_comp for next hit.
     Capture alt_comp, alt_idx, alt_ctr, alt_useful
     (u_eff of alternate table), alt_tgt.
     If prm_comp == 1 or no primary hit: alt_comp = 0.
     When alt_comp == 0: alt_* fields set to zero.

  e. Allocation candidate scan.
     Scan t = prm_comp+1 to 5. Select the first table
     with u_eff[t] == 0. Set alc_comp to that table
     index. Capture alc_idx from tbl_idx_hash_p0[t][s]
     and alc_tag from tbl_tag_hash_p0[t][s].
     If no qualifying table found: alc_comp = 0.
     When prm_comp == 5: alc_comp = 0 (no longer table).
     See ittage_cntrl_alloc_rules.md for full rules.

  f. UAON mux.
     Per ittage_cntrl_uaon_update_rules.md component
     selection logic:
       if prm_ctr != 3'b000:
         select primary. ittage_using_primary = 1.
       else if uaon[s] >= IT_UAON_THRES:
         select alternate. ittage_using_primary = 0.
         ittage_use_alt_on_na = 1.
       else:
         select primary. ittage_using_primary = 1.
     Final target is prm_tgt or alt_tgt per selection.
     ittage_pred_strong per Binding Decision 5.

     Assign all fields defined in bp_structs_pkg.sv.
     ittage_hit, branch_id from ittage_pred_inp_p0[s].
