## Hypothesis
ittage_cntrl.sv prediction path can be implemented as a pure
controller with no SRAM, following the structural patterns of
tage_cntrl.sv, with behavioral differences in provider scan
direction, CTR semantics, target handling, and no-consecutive-
table allocation constraint explicitly bound below.

## Background
ittage_cntrl.sv is the controller for the ITTAGE indirect
branch target predictor. It receives per-table hit, target,
and control bit outputs from five ittage_table instances
(IT1-IT5) and drives per-table update signals back. The
ittage_table instances are instantiated above in ittage.sv,
not here. ittage_cntrl owns all UAON counter state, epoch
and aging counter state, and prediction metadata assembly.

This task implements the prediction path only. Update path
ports are declared and all update outputs are driven to zero.
The update path is added in the follow-on task BP-034b.

tage_cntrl.sv is provided as structural reference. Use it
for signal organization, always_comb block style, per-slot
replication pattern, and p2 flop structure. The behavioral
differences from TAGE are in Binding Previous Decisions.

## Binding Previous Decisions

1. ittage_cntrl is a pure controller. No bw_ram, sram_init,
   or ittage_table instances. Those are in ittage.sv.

2. tage_cntrl.sv is structural reference only. Do NOT copy:
   - T0 base table handling. There is no IT0 in ITTAGE.
   - Direction CTR logic. CTR is confidence, not direction.
   - TAGE allocation logic. ITTAGE has the no-consecutive-
     table constraint per ittage_cntrl_alloc_rules.md.
   - TAGE type names. All types are ittage_* and IT_*.

3. Per-table signals from the five ittage_table instances
   are input ports on ittage_cntrl, arrayed over table
   index [0:IT_NUM_TABLES-1]. Index 0 is the IT0 placeholder
   and is never driven by ittage.sv. Valid indices are 1-5.

4. CTR is a confidence counter, not a direction counter.
   3'b000 = null (newly allocated, no confidence).
   3'b011 and 3'b100 are the weak boundary states.
   not_null = (ctr != 3'b000). Used internally in UAON mux.
   not_null is NOT the same as ittage_pred_strong.

5. ittage_pred_strong = (provider_ctr != 3'b011) AND
   (provider_ctr != 3'b100). Provider is whichever component
   the UAON mux selected. Source: ittage_cntrl_decisions.md
   SS Decoration flags.

6. UAON mux selection uses the not_null intermediate:
   not_null = (prm_ctr != 3'b000).
   If not_null: select primary.
   Else if uaon_r[s] >= IT_UAON_WIDTH'(IT_UAON_THRES):
     select alternate.
   Else: select primary.
   Source: ittage_cntrl_uaon_update_rules.md
   SS Component Selection During Prediction.

7. UAON registers: two 4-bit saturating counters,
   uaon_r[0] and uaon_r[1], one per prediction slot.
   Reset value: IT_UAON_THRES. Read during prediction.
   Write side driven to zero in this task; added in BP-034b.

8. Epoch and aging registers per slot:
   lcl_epoch_0, lcl_epoch_1: logic [1:0]. Reset to 0.
   lcl_aging_interval_0, lcl_aging_interval_1: logic [31:0].
   Reset to the current value of ittage_aging_interval input.
   Interval decrements on each ittage_pred_rdy_p2[s]
   assertion when ittage_enable_aging is asserted. When
   interval reaches 1 and pred_rdy fires: reload interval
   from ittage_aging_interval, increment epoch (2b wrapping).

9. IT_AGE_EPOCH in ittage_cntrl_use_update_rules.md refers
   to lcl_epoch_N for the active slot. It is NOT a package
   parameter. The u_eff formula uses this local register.
   EPOCH and USEFUL are extracted from the provider entry's
   cntrl_bits_p1 EPC and USE fields respectively.

10. cntrl_bits_p1 field layout is defined in ittage_table.sv.
    The localparam CNTRL_BITS_WIDTH and field boundary
    constants CB_CTR_H, CB_USE_H, CB_EPC_H, CB_TGT_H must
    be declared in ittage_cntrl using identical formulas.
    Verify against ittage_table.sv before using in any
    field extraction. Tag bits are not in cntrl_bits_p1;
    tag comparison and hit are resolved inside ittage_table.

11. Allocation candidate scan follows
    ittage_cntrl_alloc_rules.md exactly. Source for
    ittage_alc_idx is tbl_idx_hash_p0[alc_comp][s] --
    pre-hashed index from the candidate table output port.
    No rehashing at update time.
    Source for ittage_alc_tag is tbl_tag_hash_p0[alc_comp][s].
    ittage_alc_comp = 0 (IT0 sentinel) when no candidate.

12. p2 flop: all ittage_pred_meta_t fields flopped on posedge
    clk. ittage_pred_rdy_p2[s] is ittage_pred_val_p0[s]
    delayed one cycle via a registered flag. All meta fields
    reset to zero.

13. Per-slot logic is written as explicit slot 0 and slot 1
    always_comb blocks. No generate loops over NUM_PRED_SLOTS.

14. All update output ports to ittage_table instances are

    BP-034b reads this file as its starting point.

15. ittage_rdy output: declared and driven to 1'b0.
