## Specific Requirements

### Step 1: Read before write

Read tage_interfaces.md in full. Read tage.sv port
list. Read bp_structs_pkg.sv for tage_pred_inp_t,
tage_pred_meta_t, tage_upd_inp_t, tage_upd_inp_t.
Read tb_tage.sv for DUT instantiation pattern and
clock/reset sequence only. Do not copy test cases
from tb_tage.sv.

### Step 2: Testbench skeleton

Create tb_tage_tasks.sv. Module name is tb. Include:
- Clock and reset generation. Clock period 10ns.
  Reset held low for 4 cycles then deasserted.
- DUT instantiation: tage #(.NUM_PRED_SLOTS(2)).
- Wait for tage_rdy before beginning any test.
- Top-level error counter: integer total_errors = 0.
- Final pass/fail report on total_errors.
- +TAGE_FAST_INIT=1 passed via Makefile PLUSARGS.

### Step 3: Enable integers

Declare one plain integer enable per task group at
the top of the initial block. All set to 1 by default.
Names must match the task group they gate:

  integer en_ram_access    = 1;
  integer en_ctr_pred      = 1;
  integer en_ctr_upd       = 1;
  integer en_alloc         = 1;
  integer en_round_trip    = 1;

### Step 4: Console formatting utility tasks

Implement the following utility tasks. These are used
by all domain tasks for consistent output formatting.

  task tb_info(input string msg);
  task tb_warn(input string msg);
  task tb_error(input string msg,
                inout integer errs);

tb_error increments errs and prints the message with
an ERROR prefix. tb_info and tb_warn print with INFO
and WARN prefixes respectively. Include simulation
time in all output: $time.

### Step 5: RAM access tasks

Implement tasks for setting and reading individual
field values in TAGE table entries. These tasks set
initial conditions for test cases without depending
on hash behavior.

  task tage_ram_write(
    input int             tbl,
    input int             idx,
    input tage_ram_entry_t entry
  );

  task tage_ram_read(
    input  int              tbl,
    input  int              idx,
    output tage_ram_entry_t entry
  );

Define tage_ram_entry_t as a packed struct local to
the testbench containing: valid, ctr, use, epc, tag
fields at MAX_* widths from bp_defines_pkg.sv.

These tasks use hierarchical references to reach
bw_ram mem arrays inside tage_table instances. Read
tage.sv generate block structure to determine the
correct hierarchical path for T1-T4. T0 (tage_bim)
uses a separate path -- include a tbl==0 case.

### Step 6: Prediction request task

  task tage_predict(
    input  tage_pred_inp_t inp[0:1],
    input  logic [1:0]     val,
    output tage_pred_meta_t meta[0:1],
    output logic [1:0]      rdy,
    inout  integer          errs
  );

Drive tage_pred_val_p0 and tage_pred_inp_p0. Wait
for tage_pred_rdy_p2 to assert. Capture
tage_pred_meta_p2. Deassert valid. Report timeout
via tb_error if rdy does not assert within 16 cycles.

### Step 7: Update request task

  task tage_update(
    input tage_upd_inp_t inp[0:1],
    input logic [1:0]    val,
    inout integer        errs
  );

Drive tage_upd_val_u0 and tage_upd_inp_u0 for one
cycle. Deassert. Wait one cycle for pipeline to
settle. Report timeout via tb_error if tage_upd_rdy
does not indicate ready within 16 cycles.

### Step 8: Prediction compare task

  task tage_check_meta(
    input tage_pred_meta_t got[0:1],
    input tage_pred_meta_t exp[0:1],
    input logic [1:0]      mask,
    input string           label,
    inout integer          errs
  );

Compare got against exp field by field for each slot
where mask bit is set. Call tb_error for each field
mismatch. label identifies the calling test case in
error output.

### Step 9: Round-trip sanity test

Implement one round-trip test as the sole test case
in this session. This test is a sanity check only
and will be discarded by the project author.

The test must:
  1. Use tage_ram_write to place a known entry into
     T1 at a chosen index with valid=1, ctr=2b'10,
     use=2b'01, tag matching a chosen PC.
  2. Drive a prediction request with that PC on
     slot 0 only.
  3. Verify tage_pred_rdy_p2 asserts.
  4. Capture tage_pred_meta_p2.
  5. Drive an update using the captured meta.
  6. Drive a second prediction with the same PC.
  7. Verify rdy asserts on the second prediction.
  8. Report pass via tb_info, any failure via
     tb_error.

This test must pass without any modifications to
tage.sv, tage_table.sv, or any other RTL file. If
it does not pass, report the failure in Results
Capture and do not modify RTL. Stop after reporting.
