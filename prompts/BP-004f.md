## Assumptions made not explicit in the prompt
- Each TC8-TC13 block calls do_reset() to obtain a clean
  memory state. The prompt did not specify whether resets
  between TCs were required, but isolation between tests
  is necessary for deterministic results.
- TC8: conf starts at 2 (not yet saturated) so the
  increment to LP_CONF_LEVEL=3 is observable.
- TC9 and TC10: update bundles are driven directly without
  first allocating an entry. lp_hit=1 (TC9) and
  pred_is_loop=1 (TC10) satisfy the write condition in the
  RTL, so no prior allocation is needed.
- TC11: PC=0x80 gives idx=8 and tag=0x0082. Tags 1..4
  are used for the 4 allocated ways. Tag 0x0082 != 1..4
  ensures the prediction is a miss, confirmed analytically.
- TC12: idx_of(0x1000) == idx_of(0x2000) == 0 was verified
  analytically before writing the test.
- TC13: alloc_entry is called first to establish the entry;
  the cond1 updates then use explicit upd_p0 bundles with
  curr_itr set to itr_max-1 and itr_max respectively.
  This avoids driving 16383 individual update cycles.

## Decisions made not explicit in the prompt
- TC8-TC13 each save fail_count delta (tc12_f0, tc13_f0)
  for PASS/FAIL display in TCs with multiple check() calls
  and intermediate clock edges (TC12, TC13). TCs with a
  single prediction lookup use inline expressions instead.
- TC11 victim setup: tags 1..4 are allocated via direct
  upd_p0.tag assignment in a for loop rather than through
  idx_of/tag_of of real PCs. This avoids having to find 4
  PCs that hash to the same index. Allocation uses
  upd_p0.idx=idx11 (from idx_of(PC_11)) directly.
- Way 1 age is lowered to 0x01 via a cond5 hit update with
  upd_p0.age=0x01. Cond5 does not override the age field,
  so the passed-through value reaches the memory write.
  This was confirmed by reading the RTL upd_entry_hit
  default assignments (line 169 of loop_pred.sv).
- TC12 uses PC_12a=0x1000 and PC_12b=0x2000 because both
  hash to idx=0 with distinct tags, allowing alloc_entry
  (which uses idx_of/tag_of internally) to be reused.
- TC9/TC10 age field: left at 0 (from upd_p0='0). Neither
  test checks age, so this is acceptable.
