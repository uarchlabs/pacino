### Defect 1: t_idx_r1 and t_tag_r1 always zero

tage_cntrl.sv declares t_idx_r1 and t_tag_r1 as 2D
arrays and reads them in the prediction logic to
populate prm_idx_p1, alt_idx_p1, alc_idx_p1, and
alc_tag_p1. These arrays are never assigned. They are
always zero.

Root cause: tage_table.sv computes idx_hash and
tag_hash internally but does not expose them as
outputs. tage_bim.sv computes its index internally
and does not expose it as an output.

Fix:

Step 1 -- tage_table.sv:
Add two output ports:
  output logic [THIS_INDEX_BITS-1:0]
    idx_hash_p0[0:NUM_PRED_SLOTS-1]
  output logic [MAX_TAG_WIDTH-1:0]
    tag_hash_p0[0:NUM_PRED_SLOTS-1]

Drive them from the existing combinational idx_hash
and tag_hash signals already computed in the module:
  assign idx_hash_p0[0] = idx_hash[0];
  assign idx_hash_p0[1] = idx_hash[1];
  assign tag_hash_p0[0] = tag_hash[0];
  assign tag_hash_p0[1] = tag_hash[1];

Do not change idx_hash or tag_hash computation.
Do not change any other logic.

Step 2 -- tage_bim.sv:
Add one output port:
  output logic [THIS_INDEX_BITS-1:0]
    idx_hash_p0[0:NUM_PRED_SLOTS-1]

Drive it from the existing index computation. The T0
index formula is:
  pc[INST_OFFSET +: THIS_INDEX_BITS]
for each slot. This is already computed inside
tage_bim.sv. Expose it as a combinational output.
Do not add a tag output -- T0 has no tag.
Do not change any other logic.

Step 3 -- tage_cntrl.sv:
Add input ports to receive the new outputs from
each table instance. The port arrays must match
the existing t_idx_r1 and t_tag_r1 dimensions:
  input logic [MAX_IDX_WIDTH-1:0]
    t_idx_p0[0:TAGE_NUM_TABLES-1][0:NUM_PRED_SLOTS-1]
  input logic [MAX_TAG_WIDTH-1:0]
    t_tag_p0[0:TAGE_NUM_TABLES-1][0:NUM_PRED_SLOTS-1]

Replace the existing undriven t_idx_r1/t_tag_r1
register stage with a single always_ff that registers
t_idx_p0 and t_tag_p0 into t_idx_r1 and t_tag_r1
on posedge clk. Reset to zero. Place this always_ff
in the valid_pipe_ff region or immediately after it.
No other changes to prediction logic -- all reads of
t_idx_r1 and t_tag_r1 remain unchanged.

T0 has no tag output. t_tag_p0[0][s] must be driven
to zero in tage.sv (see Step 4). t_tag_r1[0][s] will
therefore remain zero, which is correct -- T0 has no
tag and tage_cntrl never uses t_tag_r1[0].

Step 4 -- tage.sv:
Add interconnect wires:
  logic [MAX_IDX_WIDTH-1:0]
    w_idx_p0[0:TAGE_NUM_TABLES-1][0:NUM_PRED_SLOTS-1];
  logic [MAX_TAG_WIDTH-1:0]
    w_tag_p0[0:TAGE_NUM_TABLES-1][0:NUM_PRED_SLOTS-1];

Wire T0 index from tage_bim idx_hash_p0 outputs,
zero-extended to MAX_IDX_WIDTH:
  assign w_idx_p0[0][0] =
    MAX_IDX_WIDTH'(u_tage_bim.idx_hash_p0[0]);  -- NO
Instead add idx_hash_p0 to the tage_bim port list
and wire it to w_idx_p0[0]. Drive w_tag_p0[0][s]
to zero for both slots (T0 has no tag).

Wire T1-T4 index and tag from tage_table idx_hash_p0
and tag_hash_p0 outputs inside the generate loop.
Zero-extend to MAX_IDX_WIDTH and MAX_TAG_WIDTH
respectively.

Add t_idx_p0 and t_tag_p0 to the tage_cntrl
instantiation port map.

### Defect 2: T0 prm_ctr mis-extracted

In tage_cntrl.sv the T0 CTR is extracted at two
sites in the pred_logic always_comb block.

The cntrl_bits_p1 layout for T0 as assembled in
tage.sv is:
  {5'b0, bim_cntrl_p1[s], 1'b1}
  bit[0]   = VAL = 1
  bit[2:1] = CTR[1:0]
  bit[7:3] = 0

The current extraction:
  t_cntrl_bits_p1[0][s][TAGE_T0_CTR_BITS-1:0]
evaluates to [1:0] = {CTR[0], VAL}. This is wrong.

The correct extraction is [TAGE_T0_CTR_BITS:1]
which evaluates to [2:1] = CTR[1:0].

Fix both sites in the pred_logic always_comb block:

Site 1 -- primary T0 default:
  Change:
    prm_ctr_p1[s] = MAX_CTR_WIDTH'(
      t_cntrl_bits_p1[0][s][TAGE_T0_CTR_BITS-1:0]);
  To:
    prm_ctr_p1[s] = MAX_CTR_WIDTH'(
      t_cntrl_bits_p1[0][s][TAGE_T0_CTR_BITS:1]);

Site 2 -- alternate T0 default:
  Change:
    alt_ctr_p1[s] = MAX_CTR_WIDTH'(
      t_cntrl_bits_p1[0][s][TAGE_T0_CTR_BITS-1:0]);
  To:
    alt_ctr_p1[s] = MAX_CTR_WIDTH'(
      t_cntrl_bits_p1[0][s][TAGE_T0_CTR_BITS:1]);

No other changes to CTR logic.
