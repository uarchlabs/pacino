## Background
The loop predictor is the second s1-stage predictor in the
BP cluster. It detects backward branches with constant
iteration counts and predicts loop exit. It overrides uBTB
at s1 when pred_is_loop=1 and confidence is at maximum.
It does not participate in the s2/s3 override chain.

The design follows the Seznec TAGE-SC-L loop predictor
mechanism. No read-modify-write across the predictor. All
fields needed for update are computed at prediction time
and returned in lp_pred_t. The update path uses these
stored fields directly.

Structs lp_entry_t, lp_pred_t, lp_upd_t and all LP_*
parameters are already in frontend/branch_predictor/rtl/bp_pkg.sv from BP-004a.

## Binding Previous Decisions
- bp_br_type_e is the single branch type encoding project-wide.
- File-scope wildcard import: import bp_pkg::*; before module

- lp_upd_t is flat. No sub-struct embedding.
- Backward branch filter: allocate only when

## Specific Requirements

1. Read all context files before writing any code.

2. Implement frontend/branch_predictor/rtl/loop_pred.sv. Parameters (all overridable):
     LP_TBL_ENTRIES = 256
     LP_TBL_WAYS    = 4
     LP_TAG_BITS    = 14
     LP_ITR_BITS    = 14
     LP_CNF_BITS    = 2
     LP_AGE_BITS    = 8
     LP_N_SETS      = LP_TBL_ENTRIES / LP_TBL_WAYS
     LP_IDX_BITS    = $clog2(LP_N_SETS)
     LP_CONF_LEVEL  = (1 << LP_CNF_BITS) - 1
     NUM_PRED_SLOTS = 1

   Ports:
     clk          : input  logic
     rstn         : input  logic
     pred_pc_p0   : input  logic [VA_WIDTH-1:0]
     pred_valid_p0: input  logic
     pred_p0      : output lp_pred_t
     upd_p0       : input  lp_upd_t
     upd_valid_p0 : input  logic

3. Index and tag derivation -- use these exact functions:

   function automatic logic [LP_IDX_BITS-1:0]
       idx_of(input logic [VA_WIDTH-1:0] pc);
     logic [VA_WIDTH-1:0] x;
     x = pc ^ (pc >> 1) ^ (pc >> 4);
     return x[LP_IDX_BITS-1:0];
   endfunction

   function automatic logic [LP_TAG_BITS-1:0]
       tag_of(input logic [VA_WIDTH-1:0] pc);
     logic [VA_WIDTH-1:0] x;
     x = pc ^ (pc >> 6) ^ (pc >> 12);
     return x[LP_TAG_BITS-1:0];
   endfunction

4. Prediction pipeline (combinational, registered at s1):
   - Compute req_idx and req_tag from pred_pc_p0.
   - Read all four ways at req_idx.
   - Find matching valid way (tag match). If hit and
     conf == LP_CONF_LEVEL: pred_is_loop=1.
   - Predict taken if curr_itr < past_itr.
   - Predict not-taken (exit) if curr_itr == past_itr.
   - If miss or conf < LP_CONF_LEVEL: pred_is_loop=0,
     pred_taken=0.
   - Compute victim using priority policy in step 5.
   - Output all lp_pred_t fields registered at end of s1.

5. Victim selection policy (implement as priority logic,
   derive clean RTL from this description 
   Priority 1: lowest-indexed invalid way.
   Priority 2: if all ways valid, lowest-indexed way
               with age == 0.
   Priority 3: if all ways valid and no zero-age way,
               select the way with age less than way[0];
               check way[1] first, then way[2], then
               way[3]; default to way[0] if none qualify.

6. Update logic (synchronous, driven by upd_valid_p0):
   Use upd_p0 fields directly -- no re-read of the table.

   On taken branch hit (upd_valid_p0 && upd_p0.pred_is_loop
   && upd_p0.actual_taken):
   - Increment curr_itr (saturate at LP_ITR_BITS max).
   - If curs_v: increment curs (saturate).

   On not-taken exit, correct (upd_valid_p0 &&
   upd_p0.pred_is_loop && !upd_p0.actual_taken &&
   upd_p0.curr_itr == upd_p0.past_itr):
   - Increment conf (saturate at LP_CONF_LEVEL).
   - Copy curr_itr to past_itr. Reset curr_itr to 0.
   - Reset age to maximum.

   On not-taken exit, wrong (upd_valid_p0 &&
   upd_p0.pred_is_loop && !upd_p0.actual_taken &&
   upd_p0.curr_itr != upd_p0.past_itr):
   - Reset conf to 0.
   - Copy curr_itr to past_itr. Reset curr_itr to 0.

   On mispredicted exit (upd_valid_p0 &&
   upd_p0.pred_is_loop && !upd_p0.pred_taken &&
   upd_p0.actual_taken):
   - Reset conf to 0. Reset curr_itr to 0.

   On miss with backward branch filter (upd_valid_p0 &&
   !upd_p0.pred_is_loop && upd_p0.actual_taken &&
   upd_p0.target < upd_p0.pc):
   - Allocate at victim way from upd_p0.victim.
   - Initialize: past_itr=0, curr_itr=1, conf=0,
                 age=max, v=1, curs=0, curs_v=0.
   - Write tag from upd_p0.tag.

7. Add sim_loop target to Makefile following the pattern
   of sim_ubtb. Include -Wno-VARHIDDEN on sim_loop only.
   Update the all target to include sim_loop.

8. Write frontend/branch_predictor/tb/tb_loop_pred.sv with TC1-TC12. All testcases
   self-checking with PASS/FAIL output.

   TC1:  Miss returns pred_is_loop=0.
   TC2:  Allocate on backward branch miss. Verify entry
         written at correct index and way.
   TC3:  Forward branch miss does not allocate (backward
         branch filter). Verify table unchanged.
   TC4:  Hit with conf below LP_CONF_LEVEL returns
         pred_is_loop=0.
   TC5:  Confidence builds to LP_CONF_LEVEL after correct
         exit predictions.
   TC6:  Trusted prediction -- curr_itr < past_itr
         predicts taken.
   TC7:  Trusted prediction -- curr_itr == past_itr
         predicts not-taken (exit).
   TC8:  Correct exit increments conf, copies curr_itr
         to past_itr, resets curr_itr, resets age to max.
   TC9:  Wrong exit resets conf to 0, copies curr_itr
         to past_itr, resets curr_itr.
   TC10: Mispredicted exit (predicted NT, actually T)
         resets conf and curr_itr.
   TC11: Victim selection -- all ways valid, verify
         priority policy selects expected way.
   TC12: Way conflict -- two PCs mapping to same set,
         verify independent tracking.
   TC13: curr_itr saturates at LP_ITR_BITS max, no
         overflow.
