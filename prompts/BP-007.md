## Hypothesis
A single parameterized tage_table module instantiated five
times in tage.sv covers all TAGE tables cleanly. Two bw_ram
instances per table support dual prediction slots without
conflating the bw_ram BANKS parameter with the slot concept.

## Background
Each TAGE table (T0-T4) contains two independent RAMs: one
for prediction slot 0 and one for prediction slot 1. This
mirrors the dual prediction architecture from bp_cluster.md.
The two RAMs are not banks in the bw_ram sense; they are
separate instances. bw_ram is instantiated with BANKS=1 for
each.

tage_table has no knowledge of TAGE-specific fields (valid,
tag, ctr, useful). It is a generic dual-RAM storage wrapper.
All field interpretation is done by tage_cntrl.

Entry layout (enforced by tage_cntrl, not this module):
  T0:     DATA_WIDTH=2.  Bits: ctr[1:0].
  T1-T4:  DATA_WIDTH=14. Bits [13:6]=tag, [5:4]=useful,
          [3:1]=ctr, [0]=valid.
  Control bits (valid, ctr, useful) are on the LSBs.
  sram_init writes INIT_VAL=0 to all addresses. Zero in the
  valid bit means no valid entry. This is safe for all tables.

sram_init is shared across all tables in tage.sv. It drives
the cs, wr, waddr, wdata signals into all table instances
simultaneously during initialization. tage_table exposes
init_cs, init_wr, init_waddr, init_wdata ports for this
purpose. The init ports connect to both internal RAM
instances (both slots initialized identically).

After init completes (sram_init.ready asserts), normal
predict and update ports take over. tage_cntrl gates predict
reads and update writes behind the ready signal.

Read path: 1-cycle latency (bw_ram registered address,
combinational output). Address presented at p0, data valid
at p1.

Write path: 1-cycle latency (bw_ram registered write).
Update writes are presented by tage_cntrl post-execute.

## Binding Previous Decisions
- Two bw_ram instances per table, one per prediction slot.
- bw_ram instantiated with BANKS=1 on each instance.
- bw_ram port: wen_n active-low. wen_n=1 means no write.
- bw_ram port: bweb_n active-low bit-write-enable.
  bweb_n=0 on a bit enables write to that bit.
  For full-word writes: bweb_n = '0 (all zeros).
- Read path: addr registered in bw_ram, dout combinational
  from registered address. 1 cycle latency.
- Control bits on LSBs. INIT_VAL=0 safe for all tables.
- sram_init drives init ports; normal ports used after ready.
- Port naming: <signal>_<slot>_<pipestage>.
  Init ports carry no slot or pipestage suffix.
- Package imports: none required in this module.
  tage_table has no bp package dependencies.

## Specific Requirements

### Module name and file
tage_table.sv

### Parameters
  ENTRIES    : int  -- number of entries. Default 2048.
  DATA_WIDTH : int  -- entry width in bits. Default 14.
  ADDR_BITS  : int  -- $clog2(ENTRIES). Derived, do not
                       override.

### Ports

  // clock (no reset -- bw_ram has no reset)
  input  logic                    clk

  // init interface (driven by sram_init in tage.sv)
  input  logic                    init_cs
  input  logic                    init_wr
  input  logic [ADDR_BITS-1:0]    init_waddr
  input  logic [DATA_WIDTH-1:0]   init_wdata

  // predict read interface -- slot 0
  input  logic [ADDR_BITS-1:0]    rd_addr_0_p0
  output logic [DATA_WIDTH-1:0]   rd_data_0_p1

  // predict read interface -- slot 1
  input  logic [ADDR_BITS-1:0]    rd_addr_1_p0
  output logic [DATA_WIDTH-1:0]   rd_data_1_p1

  // update write interface -- slot 0
  input  logic                    wr_en_0_u0
  input  logic [ADDR_BITS-1:0]    wr_addr_0_u0
  input  logic [DATA_WIDTH-1:0]   wr_data_0_u0
  input  logic [DATA_WIDTH-1:0]   wr_mask_0_u0  // 1=write bit

  // update write interface -- slot 1
  input  logic                    wr_en_1_u0
  input  logic [ADDR_BITS-1:0]    wr_addr_1_u0
  input  logic [DATA_WIDTH-1:0]   wr_data_1_u0
  input  logic [DATA_WIDTH-1:0]   wr_mask_1_u0  // 1=write bit

### bw_ram instantiation detail

Instantiate bw_ram twice: ram_s0 (slot 0) and ram_s1
(slot 1). Each with ENTRIES=ENTRIES, WIDTH=DATA_WIDTH,
BANKS=1.

Since BANKS=1, BANK_BITS=$clog2(1)=0. Verilator will warn
on a zero-width port. Tie bank_addr to 1'b0 on both
instances.

Write control mapping:
  During init (init_cs and init_wr asserted):
    wen_n   = 1'b0        (write enabled)
    addr    = init_waddr
    din     = init_wdata
    bweb_n  = '0          (all bits enabled)
  During normal update (wr_en_N_u0 asserted):
    wen_n   = ~wr_en_N_u0
    addr    = wr_addr_N_u0
    din     = wr_data_N_u0
    bweb_n  = ~wr_mask_N_u0  (mask: 1=write, bweb_n: 0=write)
  Init takes priority over normal update when both asserted.
  Use a priority mux in always_comb to select.

Read control mapping:
  addr    = rd_addr_N_p0  (presented each cycle)
  dout    -> rd_data_N_p1 (1-cycle latency from bw_ram)

The read address port of bw_ram is shared with the write
address port (single addr input). During a write cycle the
read address is also updated. tage_cntrl must not rely on
read data being stable during a write to the same address
(read-during-write is undefined per bw_ram spec).

### Testbench requirements
Module name inside testbench: tb
File name: tb_tage_table.sv
Instantiate with ENTRIES=16, DATA_WIDTH=14, for simulation
speed. Use ENTRIES=16 so ADDR_BITS=4.

TC1 (init sequence):
  Assert init_cs, init_wr. Walk init_waddr from 0 to 15
  with init_wdata=14'h0. After completing the walk, deassert
  init_cs and init_wr. Read back each address on slot 0
  and slot 1 read ports. Verify all entries read as 14'h0.

TC2 (slot 0 write, slot 0 read):
  Write 14'h1234 to address 5 via wr_en_0, wr_addr_0=5,
  wr_data_0=14'h1234, wr_mask_0=14'h3FFF (all bits).
  Next cycle: present rd_addr_0=5.
  Cycle after: verify rd_data_0 = 14'h1234.

TC3 (slot 1 write, slot 1 read):
  Write 14'h0ABC to address 10 via slot 1 write ports.
  Verify via slot 1 read ports.

TC4 (slot independence):
  Write different values to address 7 via slot 0 and slot 1.
  Verify slot 0 reads its own value and slot 1 reads its own.
  Confirm the two RAMs are independent.

TC5 (partial write via mask):
  Write 14'h3FFF to address 3 (all bits set).
  Then write with wr_mask=14'h0001 (bit 0 only), wr_data=0.
  Verify result is 14'h3FFE (only bit 0 cleared).

TC6 (init priority over write):
  Assert init_cs, init_wr, init_waddr=0, init_wdata=14'h0.
  Simultaneously assert wr_en_0, wr_addr_0=0,
  wr_data_0=14'h3FFF. Verify that after the cycle the entry
  reads as 14'h0 (init wins).
