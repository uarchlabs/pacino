## Background
The loop predictor is the second s1-stage predictor in
the BP cluster. It detects backward branches with
constant iteration counts and predicts loop exit. It
overrides uBTB at s1 when pred_is_loop=1 and confidence
is at maximum. It does not participate in the s2/s3
override chain.

The design follows the Seznec TAGE-SC-L loop predictor
mechanism. No read-modify-write across the predictor.
All fields needed for update are computed at prediction
time and returned in lp_pred_t. The update path uses
these stored fields directly.

All structs (lp_entry_t, lp_pred_t, lp_upd_t) and all
LP_* parameters are already defined in bp_defines_pkg.sv
and bp_structs_pkg.sv. Do not redefine them.

---

- bp_br_type_e is the single branch type encoding

- lp_upd_t is flat. No sub-struct embedding.

     LP_ITR_BITS    = 14

     LP_AGE_BITS    = 8
     LP_N_SETS      = LP_TBL_ENTRIES / LP_TBL_WAYS
     LP_IDX_BITS    = $clog2(LP_N_SETS)
     LP_CONF_LEVEL  = (1 << LP_CNF_BITS) - 1
     NUM_PRED_SLOTS = 1

   Ports:
     clk           : input  logic
     rstn          : input  logic
     pred_pc_p0    : input  logic [VA_WIDTH-1:0]
     pred_valid_p0 : input  logic
     pred_p0       : output lp_pred_t
     upd_p0        : input  lp_upd_t
     upd_valid_p0  : input  logic

3. Index and tag derivation -- use these exact functions:

   function automatic logic [LP_IDX_BITS-1:0]
       idx_of(input logic [VA_WIDTH-1:0] pc);
     logic [VA_WIDTH-1:0] x;
     x = pc ^ (pc >> 1) ^ (pc >> 4);
     return x[LP_IDX_BITS-1:0];
   endfunction

   function automatic logic [LP_TAG_BITS-1:0]
       tag_of(input logic [VA_WIDTH-1:0] pc);
     logic [VA_WIDTH-1:0] x;
     x = pc ^ (pc >> 6) ^ (pc >> 12);
     return x[LP_TAG_BITS-1:0];
   endfunction

4. Prediction pipeline (combinational, registered at s1):
   - Compute req_idx and req_tag from pred_pc_p0.
   - Read all four ways at req_idx.
   - Find matching valid way (tag match). If hit and
     conf == LP_CONF_LEVEL: pred_is_loop=1.
   - Predict taken if curr_itr < past_itr.
   - Predict not-taken (exit) if curr_itr == past_itr.
   - If miss or conf < LP_CONF_LEVEL: pred_is_loop=0,
     pred_taken=0.
   - Compute victim using priority policy in step 5.
   - Output all lp_pred_t fields registered at end of s1.

5. Victim selection (priority logic):
   Priority 1: lowest-indexed invalid way.
   Priority 2: if all ways valid, lowest-indexed way
               with age == 0.
   Priority 3: if all ways valid and no zero-age way,
               select the way with age less than way[0];
               check way[1] first, then way[2], then
               way[3]; default to way[0] if none qualify.

6. Update logic (synchronous, driven by upd_valid_p0):
   Use upd_p0 fields directly -- no re-read of the table.

   On taken branch hit (upd_valid_p0 &&
   upd_p0.pred_is_loop && upd_p0.actual_taken):
   - Increment curr_itr (saturate at LP_ITR_BITS max).
   - If curs_v: increment curs (saturate).

   On not-taken exit, correct (upd_valid_p0 &&
   upd_p0.pred_is_loop && !upd_p0.actual_taken &&
   upd_p0.curr_itr == upd_p0.past_itr):
   - Increment conf (saturate at LP_CONF_LEVEL).
   - Copy curr_itr to past_itr. Reset curr_itr to 0.
   - Reset age to maximum.

   On not-taken exit, wrong (upd_valid_p0 &&
   upd_p0.pred_is_loop && !upd_p0.actual_taken &&
   upd_p0.curr_itr != upd_p0.past_itr):
   - Reset conf to 0.
   - Copy curr_itr to past_itr. Reset curr_itr to 0.

   On mispredicted exit (upd_valid_p0 &&
   upd_p0.pred_is_loop && !upd_p0.pred_taken &&
   upd_p0.actual_taken):
   - Reset conf to 0. Reset curr_itr to 0.

   On miss with backward branch filter (upd_valid_p0 &&
   !upd_p0.pred_is_loop && upd_p0.actual_taken &&
   upd_p0.target < upd_p0.pc):
   - Allocate at victim way from upd_p0.victim.
   - Initialize: past_itr=0, curr_itr=1, conf=0,
                 age=max, v=1, curs=0, curs_v=0.
   - Write tag from upd_p0.tag.

## Assumptions made not explicit in the prompt
1. hit_entry defaults to mem[req_idx][0] on true miss.
   Fields propagate to pred_comb but the consumer ignores
   them when pred_is_loop=0.
2. curs and curs_v are NOT modified by the mispredicted
   exit or wrong exit update paths. The spec is silent on
   these cases; keeping them unchanged is the safe default
   pending LI4 resolution.
3. Mispredicted exit (cond 4: !pred_taken && actual_taken)
   is checked before taken branch hit (cond 1: actual_taken)
   in the always_comb update block because cond 4 is a
   subset of cond 1 and takes priority.
4. hit_entry.cnf == {LP_CNF_BITS{1'b1}} is used instead
   of == LP_CONF_LEVEL to avoid comparing a packed field
   against an int parameter (width mismatch risk).

## Decisions made not explicit in the prompt
1. Two always_comb blocks: one for prediction, one for
   update entry computation. Keeps concerns separate and
   avoids inferred latches.
2. Asynchronous active-low reset (posedge clk or negedge
   rstn) for both mem array and pred_p0 register. All
   memory entries cleared to '0 on reset (v=0).
3. Memory read is purely combinational from the register
   array. No additional pipeline register for mem read.
4. Loop variable int w converted to LP_WAY_BITS width
   using w[LP_WAY_BITS-1:0] bit-selection (avoids
   WIDTHTRUNC on int->logic assignment).
5. Literal way indices 1/2/3 in victim priority 3 use
   LP_WAY_BITS'(N) cast (localparam from package, N=1,2,3).
   If Verilator rejects this form, fallback is 2'd1/2/3.
6. Update write address: upd_p0.idx and upd_p0.way for
   hit path; upd_p0.idx and upd_p0.victim for allocation.
   No table re-read on update path.

## Deferred Work

1. Lint run deferred to BP-004d. -Wno-VARHIDDEN must be
   added to both sim_loop and lint targets in the Makefile.
   The CLAUDE.md currently says "sim targets only" but the
   lint target will also trigger VARHIDDEN.

2. STRUCTURAL GAP -- lp_pred_t/lp_upd_t missing lp_hit:
   The spec update conditions all require pred_is_loop=1
   to update the existing entry. pred_is_loop=1 requires
   conf==LP_CONF_LEVEL. conf is only incremented at correct
   exit (cond 2) which also requires pred_is_loop=1. This
   is a chicken-and-egg: conf can never advance from 0 to
   LP_CONF_LEVEL from a cold start. The fix (Seznec's
   original design) is to update curr_itr on ALL hits
   regardless of confidence, gating only the PREDICTION on
   confidence. This requires distinguishing true miss from
   low-confidence hit. bp_loop_meta_t.lp_hit (already in
   bp_structs_pkg.sv) expresses the intent, but lp_pred_t
   does not output lp_hit, and lp_upd_t does not carry it.
   Resolution: add lp_hit to lp_pred_t and lp_upd_t; add
   a learning update condition (lp_hit && !pred_is_loop)
   that increments curr_itr without requiring trusted hit.
   The RTL is implemented per the spec as given; this gap
   must be resolved before the loop predictor can learn
   iteration counts from a cold start.

   Testbench deferred to BP-004d or a dedicated session.

4. curs/curs_v rollback policy on redirect is TBD (LI4
   in loop_pred_interfaces.md). The curs field is
   maintained but the policy for restoring it after a
   mispredict flush is not yet defined.
