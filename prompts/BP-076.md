## Hypothesis
sc_brimli is sc_table with one difference: the prediction index is
get_br_imli_idx (over PC[15:6], sc_phr_p2, br_imli, br_imli_mode)
instead of sc_idx_hash. Entry, update path, tbl_ri path, and read
latency are identical to sc_table. A single prompt can produce the
module, its unit testbench, and the sim/lint targets.

## Background
sc_brimli implements SC table ST4. The entry is a single signed
counter (THIS_CTR_WIDTH), no tag/USE/EPC/valid/allocation, same as
sc_table. ST4 differs from ST0-ST3 only in the index function and in
geometry (SC_TBL_ENTRIES[4]=1024, SC_TBL_IDX[4]=10).

sc_brimli is the counter RAM only. The BrIMLI registers (br_imli,
bb_hist, last_back_pc) live in sc_cntrl (sc_decisions.md section 8).
br_imli is presented to sc_brimli as an input port; sc_brimli does
not maintain it. Register maintenance (sc_decisions.md section 12) is
not in this module.

Index: get_br_imli_idx (sc_table_hash_rules.md, sc_decisions.md
section 12). Inputs PC[15:6] (from inp_pc_p2[15:6]), sc_phr_p2,
br_imli, br_imli_mode. Return [9:0].

Update index is captured at predict time into sc_pred_meta.sc_upd_idx
and presented on upd_index_u0. No re-hash on update, same as
sc_table.

Port list and semantics: sc_table_interfaces.md (sc_brimli Port
List).
Test plan and conventions: sc_tb_decisions.md.
Structural and tb pattern references: sc_table.sv, tb_sc_table.sv.
