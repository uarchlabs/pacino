## Background
tage_table.sv was produced in BP-007a. Two fixes are required:

Fix 1 -- Generate loop removal:
The module uses a generate for-loop over NUM_PRED_SLOTS.
This is being replaced with two explicit unconditional blocks,
one for slot 0 and one for slot 1. NUM_PRED_SLOTS is retained
as a parameter but is always 2 for current design work.

Fix 2 -- CNTRL_BITS_WIDTH correction:
BP-007a defined:
  CNTRL_BITS_WIDTH = MAX_CTR_WIDTH+MAX_USE_WIDTH+MAX_VAL_WIDTH
This omits MAX_EPC_WIDTH. EPC is a control field per the entry
layout. The correct definition is:
  CNTRL_BITS_WIDTH = MAX_EPC_WIDTH+MAX_USE_WIDTH
                   + MAX_CTR_WIDTH+MAX_VAL_WIDTH
ALLOC_DATA_WIDTH is unchanged:
  ALLOC_DATA_WIDTH = CNTRL_BITS_WIDTH + THIS_TAG_BITS
This is now self-consistent with the entry layout.

The wider CNTRL_BITS_WIDTH changes the width of cntrl_bits_p1
and the expected values in the testbench TCs that check
cntrl_bits_p1. All TC expected values must be recomputed
against the corrected layout.

Change the CNTRL_BITS_WIDTH localparam to:
