Four causes produce the visible and latent ITTAGE unit defects.

violate. The third is a CTR prm/alt routing direction error whose

document, not by either artifact alone. The fourth, the

## Background

BP-047 triage findings (starting map, to be verified, not trusted
blindly):

- TC-USE-EPC (tb_ittage_table, 2 FAIL): use_we/epc_we in
  ittage_table.sv gate on (prm_ctr_wr & prm_match | alt_ctr_wr &
  alt_match). The test drives use_wr/epc_wr without a CTR write
  strobe, so the gate is closed and nothing writes. The
  ittage_table.sv header comment documents the co-assertion
  requirement.

- pred_s0 timing (tb_ittage_cntrl, 30 FAIL: TC-PRED-02..06 and
  TC-UPD-08): the pred_s0 task clears tbl_hit_p1 / cntrl_bits /
  pred_tgt at #1 after the p0->p1 posedge, before the p1->p2
  capture. The nba_sequent prv_alt_scan re-evaluates with zeroed
  inputs and meta_p2_reg latches zeros. Real ittage_table holds
  FF outputs stable for the full p1 period; the testbench does
  not model this.

- CTR prm/alt inversion (ittage_cntrl.sv g_ctr_upd, 14 FAIL:
  TC-UPD-01..05): the test expects, for using_primary=1, that
  alt_ctr_wr fires with alt_ctr data; the RTL fires prm_ctr_wr
  with prm_ctr data. Which is correct is decided in Requirement 1
  before either is changed. CTR in ITTAGE is a confidence counter
  (per bp_cluster decisions), not a direction counter; TAGE/ITTAGE
  CTR-semantics contamination is a known recurring error class.

- t_alc_index_u0 source defect (ittage_cntrl.sv): driven from the
  primary update index rather than the allocation index field.
  TC-UPD-06 passes despite this because its fixture sets
  ittage_prm_idx == ittage_alc_idx, so both sources carry the
  same value. The passing test cannot detect the defect. The fix
  requires a test where the two indexes differ.

- pACT = -- (no primary CTR action)
- aACT = INC or DEC
- Governing strobe: alt_ctr_wr
- Data source: ittage_alt_ctr
- Gate component: ittage_alt_comp != 0

UP=1 (primary is provider), rows 18-33:
- pACT = INC or DEC
- aACT = -- (no alternate CTR action)
- Governing strobe: prm_ctr_wr
