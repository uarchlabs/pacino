## Background
TAGE is a tagged geometric history length predictor providing
direction prediction for conditional branches. Five tables:
T0 is the base table (2b CTR, no tag, no useful). T1-T4 are
tagged tables (valid=1b, tag=8b, ctr=3b, useful=2b),
2 banks x 2048 entries each.

Three new structs are required before the TAGE RTL module
can be implemented:

tage_pred_inp_t: input bundle for each prediction request.
  Contains pc and branch_id. Simple at this point.

tage_pred_meta_t: prediction output and metadata. This is
  a rename of bp_tage_meta_t with branch_id added.
  bp_tage_meta_t is retained in bp_structs_pkg.sv during
  transition -- do not remove it.

tage_upd_inp_t: update input bundle. Contains
  tage_pred_meta_t captured at predict time plus
  resolved_taken and cond_mispredict from the execute
  stage.

TAGE_MAX_AWIDTH is currently derived from TAGE_T0_ENTRIES
alone. This is fragile -- it must be derived as the max
of all five per-table IDX_BITS so it remains correct when
table sizes are adjusted during performance modeling.

---

## Binding Previous Decisions
- Import order is mandatory: bp_defines_pkg first, then
  bp_structs_pkg. File-scope wildcard import before module
  declaration.
- All field widths must use named parameters from
  bp_defines_pkg. No magic numbers.
- tage_upd_inp_t embeds tage_pred_meta_t as a sub-struct.
  This is the one exception to the flat struct rule --
  the update path carries the full prediction metadata
  as a unit.
- TAGE_MAX_AWIDTH must be derived as the max of T0-T4
  IDX_BITS using nested ternary. It must not be hardcoded
  to any single table entry count.
- bp_tage_meta_t is already present and correct. Do not
  modify or remove it.
