## Hypothesis

The ITTAGE EPC write enable is gated provider-only and writes
the correct EPC value to exactly the provider table entry. The
competing hypothesis is that epc_we uses the (prm_match |
alt_match) gate that affected CTR, USE, and TGT, corrupting the
non-provider entry. A readback of only the provider cannot tell
these apart. Settle by readback of both provider and
non-provider, per provider.

## Background

EPC is a control field, written on update as a USE rider.
Enables are epc_we_s0/s1 in ittage_cntrl, consumed by the EPC
write gate in ittage_table. BP-044c changed these alongside
use_we with no positive readback test.

The provider-gating defect was found and fixed in CTR
(BP-044b), USE (BP-044c), TGT (BP-049a): each used (prm_match |
alt_match), writing both hitting tables. BP-049a corrected TGT
by splitting the strobe into per-slot provider strobes; the
table gate became (prm_tgt_wr & prm_match | alt_tgt_wr &
alt_match). BP-049a also recorded that a using_primary
table-input with partial-bit generate assignments was scheduled
incorrectly by Verilator v5.048; the split-strobe pattern is the
one that worked.

BP-050 (abandoned) reported the EPC gate already provider-only
because it piggybacks on the mutually exclusive prm_ctr_wr_u0 /
alt_ctr_wr_u0 strobes. That finding is plausible but was not
independently demonstrated: BP-050's non-provider check passed
but was never shown to fail under a defective gate, so its
discriminating power is unproven. This task reproves from
scratch and adds the fail-before demonstration.

## Binding Previous Decisions

- Provider-only write is the binding invariant. Only the
  provider table entry's EPC is written on an EPC-writing
  update, even when both hitting tables satisfy the condition.
- UP=1: primary is provider. UP=0: alternate is provider. The
  non-provider is the other hitting table.
- Readback through prediction is preferred; use direct RAM read
  where post-alloc prediction ambiguity applies (BP-049a). Seed
  via bw_write backdoor.
- If a fix is required, use the per-slot provider split-strobe
  pattern. Do not drive the EPC write from a using_primary
  table-input with partial-bit generate assignments (Verilator
  v5.048 scheduling, BP-049a).

## Decisions made not explicit in the prompt

- TC-EPC-UP1 does not call do_reset() before itself (unlike
  TC-EPC-UP0). UP=1 is controlled by seeding IT2.ctr=1 (not-null)
  and does not depend on UAON. This avoids unnecessary reset
  overhead and keeps the test self-contained by construction.
- Discriminating values: provider seeded at 2'h03, non-provider at
  2'h02, written value 0. Three-way distinct ensures any crossing
  is caught unambiguously.

  the alternate and therefore the provider. This exercises the full
