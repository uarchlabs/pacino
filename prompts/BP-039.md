  Here is what it means concretely. In SystemVerilog, @(posedge clk) suspends

  timestep — after all FFs have already sampled. So:
  
  @(posedge clk);        // waits for T_start edge; FFs sample HERE (pred_val

  pred_val_p0 = 1;       // NOW driven, but T_start posedge is already past
  @(posedge clk);        // waits for T_P1 edge; FFs sample HERE (pred_val = 1)
  pred_val_p0 = '0;      // deassert

  The flip-flops that matter — pred_val_p1, bw_ram's raddr_q, tag_hash_p1 in

  encounters. That is why the Results Capture says "the p0→p1 register captures
  it at the second edge." The first edge is the one you wait through to drive
  the signal; the second edge is the one the pipeline sees.

  delivers correct results," you first have to establish that pred_val is stable

Two cycles. Here is the pipeline broken down by posedge:

  Cycle N   : pred_val_p0=1, pc driven (after posedge N)

              tag_hash_p1 captured

              hit_p1 computed combinatorially

              -> response valid HERE

  the start of cycle N+2, two cycles after you drive pred_val_p0.

  - p1 combinatorial: RAM dout valid after raddr_q updates, hit/tag comparison

## Background
ittage.sv wraps ittage_cntrl and five ittage_table
instances (IT1-IT5). The arbitration layer (PQ, UQ,
credit arbiter) was added in BP-038. The response
buffer was removed in BP-038b. Outputs are driven
directly from ittage_cntrl.

FAST_INIT (plusarg +ITTAGE_FAST_INIT=1) initializes
all RAM entries to IT_SRAM_INIT_VALUE=0 at time zero
and drives ittage_rdy=1 immediately. With all valid
bits zero, all initial predictions produce
ittage_hit=0.

Round-trip hit methodology (no hash knowledge needed):
  Step A -- Predict with PC=A, folded_hist=B. No hit
            (all valid=0). Capture ittage_alc_comp,
            ittage_alc_idx, ittage_alc_tag from
            returned meta at p2.
  Step B -- Update with indir_mispredict=1,
            prm_comp=0 (no hit at predict time),
            alc_comp/alc_idx/alc_tag from Step A,
            resolved_target=TARGET. Allocation fires:
            prm_comp(0) < IT_NUM_TABLES-1(5) and
            alc_comp != 0.
  Step C -- Predict with same PC=A, folded_hist=B.
            Hash is deterministic -- same inputs
            produce same index. Entry valid=1, tag
            matches. Expect ittage_hit=1,
            ittage_prm_tgt=TARGET.

Arbitration parameters (from bp_defines_pkg.sv):
  ITTAGE_PQ_DEPTH=4, ITTAGE_UQ_DEPTH=2
  ITTAGE_PRED_CREDITS=2, ITTAGE_UPD_CREDITS=1
  ITTAGE_STARVE_THRESH=2

ARB-08 is testable: PRED_CREDITS=2, STARVE_THRESH=2.
After 2 pred grants with upd pending, starve_ctr=2
>= threshold=2, upd grant fires next.

ARB-07 (RB full) is N/A. RB removed in BP-038b.

### Bug 1 -- ittage_cntrl.sv: branch_id pipeline timing
Symptom: P01:bid and P02:bid0/bid1 failed. branch_id

### Bug 2 -- ittage_cntrl.sv: wrong alc_index_u0

were diagnosed. alc_index_u0 pointed to wrong address.
Root cause: upd_sel always_comb computed alc_index_u0
from ittage_prm_idx or ittage_alt_idx based on
using_primary. For a no-hit prediction prm_idx=0,

### Bug 3 -- ittage_table.sv: tbl_ri_active overrides
                              write addr/data

remained 0 after Phase B allocation write.

tbl_ri_active as the first priority condition. With
FAST_INIT=1, sram_init's state machine is still running
(state=INIT) so tbl_ri_active=1 throughout early test

din_mux forced ram_din to tbl_ri_wd=0, overriding the
valid allocation write data (alc_wd_u0).
Fix: Changed all four mux guards from tbl_ri_active to
ri_we (= tbl_ri_active & tbl_ri_wr). With fast_init,
ri_we=0 so normal addr/data paths are used.
File: rtl/core/frontend/bpu/rtl/ittage_table.sv

### Bug 4 -- ittage_cntrl.sv: ctr_upd using_primary
                              condition inverted
Symptom: TC-P04 prm_ctr=0 after a correct prediction
using the primary provider (using_primary=1).
Root cause: ctr_upd always_comb had using_primary=1
branch update ALT CTR, and using_primary=0 branch

primary (using_primary=1), the primary provider's CTR
should be updated, not the alternate.

updates alt CTR.
