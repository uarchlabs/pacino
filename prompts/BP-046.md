# Overview of task

TD #76. ITTAGE update routes the alternate provider through a
shared update index, which is wrong: primary and alternate are
different tables at different history lengths with independently
hashed indexes, and both can be written in the same cycle. Add a
separate alternate update index bus (mirrors the TAGE BP-045
fix) and rename the ittage_cntrl ports that touch the tables to
the tage_cntrl t_ convention. Escapes are NOT fixed in this task;
the existing ITTAGE failure count must be unchanged before vs
after.

## Hypothesis

ITTAGE update currently carries one update index bus shared by
primary and alternate providers. Because primary and alternate
are different tables at different history lengths with
independently hashed indexes, a simultaneous prm+alt CTR/USE
write needs two distinct indexes. Adding a separate alternate
update index bus, and applying the t_ port-naming convention,
is structural: the existing ITTAGE failure count is unchanged.

## Background

BP-045 closed the equivalent TAGE defect (TD #66). TAGE update
now uses three index buses: t_prm_upd_index_u0 (primary update),
t_alt_upd_index_u0 (alternate update), t_alc_index_u0
(allocation). ITTAGE lacks the separate alternate update index.
The ittage meta structs already carry both primary and alternate
index fields, so the source data exists; only the bus and its
routing are missing.

Separately, the ittage_cntrl ports that connect to the table
instances do not use the tage_cntrl naming convention. They
should begin with t_ to match tage_cntrl.

## Binding Previous Decisions

- Three update/alloc index buses, mirroring TAGE post-BP-045:
    t_prm_upd_index_u0  -- primary provider update index
    t_alt_upd_index_u0  -- alternate provider update index
    t_alc_index_u0      -- allocation index
  prm and alt are distinct tables and may be written the same
  cycle. They must not share an index bus.
- ittage_cntrl ports that touch the table instances use the
  tage_cntrl convention: prefix t_, slot dimension
  [0:NUM_PRED_SLOTS-1], suffix _u0 for the update path.
- Routing currency is the existing *_tbl_sel_u0 selects, as in
  TAGE. Do not introduce a new routing mechanism.
- This task does NOT fix the existing ITTAGE test escapes. The
  failure count must be identical before and after. Any change
  in failure count -- including a test that newly passes -- is
  unexpected and must be reported, not absorbed.

## Specific Requirements

Read before write. Do not modify a file before reading it.

   sim_ittage) on the unmodified baseline. Record the exact

5. Add t_alt_upd_index_u0[0:NUM_PRED_SLOTS-1] as a companion to
   the primary update index. Drive it from the alternate index
   meta field. Rename the existing primary update index bus to
   t_prm_upd_index_u0 if it does not already match.
6. In ittage.sv, route per table: the alternate table receives
   t_alt_upd_index_u0, the primary table receives
   t_prm_upd_index_u0, selected by the corresponding *_tbl_sel_u0
   comparison at the connection boundary, exactly as TAGE does
   post-BP-045.
7. Rename every ittage_cntrl port that connects to the table
   instances to the t_ convention. Update the matching
   connections in ittage.sv. Do not rename internal-only signals
   that never cross to the tables.
8. Re-run the full ITTAGE suite. The after count must equal the
   before count from step 1.
