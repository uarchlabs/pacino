## Hypothesis
sc_table is a counter-only RAM table with a combinational index
hash, a one-cycle read, a whole-word update write, and a tbl_ri
override path. A self-checking unit testbench can verify each
operation directly, using hierarchical references to the two bw_ram
instances, without sc.sv, sram_init, or sc_ready.

## Background
sc_table implements SC tables ST0-ST3. The entry is a single signed
counter (THIS_CTR_WIDTH). No tag, USE, EPC, valid bit, or
allocation. Prediction: index at p2, ctr_p3 at p3. Update: whole-word
write at upd_index_u0 gated by sc_upd_val_u0 & ctr_wr_u0. tbl_ri
override gates all writes.

Test plan and conventions: sc_tb_decisions.md.
Index hash: sc_table_hash_rules.md (sc_idx_hash).
Port semantics: sc_table_interfaces.md.
DUT: sc_table.sv.

tb_tage_table.sv is the structural pattern reference for the unit-tb
skeleton, the hierarchical RAM-content check, and the per-test enable
convention. sc_table is simpler: no tag, no useful, no allocation.

## Binding Previous Decisions
- DUT instantiated directly as u_dut, parameters fixed at build.
  One parameterized instance, ST1 (THIS_TABLE=1, hashed index,
  non-zero fold). ST0 coverage is a later elaboration, not in this
  task.
- Hierarchical paths: u_dut.u_ram_s0.mem[b][i],
  u_dut.u_ram_s1.mem[b][i]. bw_ram mem is 2D mem[BANKS][ENTRIES].
- Fast init plusarg: SC_FAST_INIT (SC-scoped). sim_sc_table_fast
  passes +SC_FAST_INIT=1; sim_sc_table drives the tbl_ri init
  sequence.
- NUM_BANKS=2. ST1: THIS_ENTRIES=512, RAM_ENTRIES=256.
  bank_addr = index[THIS_INDEX_BITS-1] = index[8];
  row = index[7:0].
- sc_pred_val_p2 is unconnected in the DUT. Tie it to a constant.
  Do not test it.
- The THIS_TABLE-range assert is not tested (unreachable for a valid
  instantiation).
- Prediction and update are phase-exclusive. Seed and read occupy
  separate cycles.

## Specific Requirements
1. Read sc_tb_decisions.md, sc_table_interfaces.md,
   sc_table_hash_rules.md, and tb_tage_table.sv before writing.
2. Module name tb, file tb_sc_table.sv. Instantiate sc_table as
   u_dut with THIS_TABLE=1 and the ST1 params from bp_defines_pkg.sv.
3. Self-checking. No manual waveform inspection. Report a PASS/FAIL
   count and exit zero on all-pass. A basic sanity path runs under
   10 seconds.
4. Per-test enable convention (sc_tb_decisions.md Test Structure):
   int verbose, int _testname per test; if (_testname != 0) invoke.
   Use if (x != 0) not if (x).
5. Init check: after init (both modes), read every entry in both
   banks via hierarchical reference; verify SC_SRAM_INIT_VALUE.
6. Prediction read / index hash: drive inp_pc_p2[s] and idx_fh_p2;
   compute exp_idx = THIS_INDEX_BITS'((pc >> INST_OFFSET) ^ fh_ext)
   in the tb (fh_ext = idx_fh_p2 for ST1); verify idx_hash_p2[s].
   Seed a counter at exp_idx, deassert write, read next cycle,
   verify ctr_p3[s] at p3.
7. Update write: drive sc_upd_val_u0[s], ctr_wr_u0[s],
   upd_index_u0[s], ctr_wd_u0[s]; verify the addressed entry holds
   ctr_wd_u0 via hierarchical reference; verify no other entry
   changes; verify no write when sc_upd_val_u0[s]=0 or
   ctr_wr_u0[s]=0.
8. Slot independence: drive slot 0 and slot 1 with different indices
   and data the same cycle; verify u_ram_s0 and u_ram_s1 each
   receive only their slot's transaction.
9. tbl_ri override: assert tbl_ri_active with a concurrent update;
   verify tbl_ri address/data/enable win and the update is
   suppressed.
10. Bank decomposition: drive index[THIS_INDEX_BITS-1]=0 and =1;
    verify the entry lands in mem[0][row] and mem[1][row].

## Assumptions made not explicit in the prompt
- inp_pc_p2 is [VA_WIDTH-1:1] (39 bits); PC literals are 39-bit to
  avoid WIDTHTRUNC. The tb's calc_idx uses the identical DUT
  expression (SC_MAX_FH cast, shift by INST_OFFSET, XOR fold, cast to
  THIS_INDEX_BITS) so tb and DUT agree numerically.
- Slow-mode tbl_ri init walks the full index range 0..THIS_ENTRIES-1;
  since bank is the index MSB this covers both banks (256 rows each).
- Reset is driven for interface completeness only; sc_table has no
  reset-driven state and bw_ram has no reset. A short reset window
  also lets the fast-init initial block settle before TC1.
- The tb reads +SC_FAST_INIT itself to decide whether to run the
  tbl_ri init sequence, matching the sc_table plusarg convention.

## Decisions made not explicit in the prompt
- -Wno-VARHIDDEN added to both sim targets, matching lint_sc_table
  (BP-075): the sc_table NUM_PRED_SLOTS parameter intentionally
  shadows bp_defines_pkg::NUM_PRED_SLOTS. No other suppressions beyond
  the project-wide VER_FLAGS and SIM_EXTRA_FLAGS.
- Requirement 6 phrased with [s]; both slots are exercised (distinct
  PCs, shared non-zero fold) for the index-hash and read-back checks.
- TC3 "no other entry changes" is proven by a full 512-entry snapshot
  and diff of slot 0, not by spot-checking a single neighbor.
