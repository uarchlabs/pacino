## Background
tage_cntrl_use_update_rules.md defines the USEFUL counter
update rules for the TAGE predictor. The USEFUL field is a
2-bit saturating counter per T1-T4 entry. It tracks whether
an entry has recently provided a correct prediction that
disagreed with the alternate provider. A non-zero USEFUL
value protects an entry from eviction. The rules specify:

- When useful_wr is asserted and when it is suppressed
  (Table 7, rows 1-6)
- Which component, index, and write-data are selected for
  the update (Useful SEL, IDX, WD columns)
- Whether to increment or decrement the effective useful
  value (Useful Action column)
- How the effective useful value u_eff is computed from
  the raw USEFUL field and the epoch counters (the age
  formula in the document)
- How lcl_aging_interval_0/1 and lcl_epoch_0/1 operate

computed ueff_p1[t][s] value in both the primary and alternate
provider scan loops.
