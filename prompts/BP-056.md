## Background

#55 is the TAGE half of the EPC write proof. The ITTAGE half
(#56, BP-050b) found the ITTAGE EPC write to be a strict subset
of the CTR write. TAGE differs: epc_we rides the USE write
enable -- HAND-FIX-001 changed both use_we and epc_we to gate
on prm_alt_match (prm_match | alt_match) in BP-044c. The
alternate-match EPC write is therefore a path the BP-044c change
enabled and that has never been exercised by readback.

BP-050 (the abandoned first ITTAGE attempt) reported the gate
provider-only but never showed the non-provider check could fail
under a defect, so its discriminating value was unproven. This
task does not repeat that error: it includes a defect-injection
step that reverts the gate and shows the alternate-match check
fails before restoring.

## Assumptions made not explicit in the prompt

1. folded_hist=0 assumed throughout (all sim_tage tests run
   at reset history state, zero global history). idx and tag
   hash functions simplify to pc-based masks only.
   Assumption: safe -- tests use the same reset+predict
   pattern as all other sim_tage tests, which also rely on
   zero global history.

2. u_eff formula: with seeded EPC=00, lcl_epoch=2'b10,
   age=2, the USE-update formula yields u_eff=0. USE INC
   result = u_eff+1 = 01; USE DEC result = clamp(u_eff-1)=
   00. This was derived empirically from simulation output
   (TC-69 T2 USE INC returned 01, not 10).

3. meta.tage_using_primary is a writable struct field in
   simulation, overrideable after the pipeline read in TC-71
   and TC-72 to force the alt-is-provider path without
   requiring a specific history pattern.

4. Slot 0 only (per Constraints). All tests seed and read
   mem[bank][row] directly, slot index [s]=0.

## Decisions made not explicit in the prompt

1. Chosen test PC = 40'h3800 throughout all five tests.
   This gives a clean idx=0x600 (bank=1, row=512) with
   tag=0x07, separating bank/row fields clearly and reusing
   the addressing derived from other tage TB tasks.

2. T3 seeded invalid (VAL=0) with a distinct EPC value to
   serve as the isolation reference. An invalid entry cannot
   be a provider or alternate match, so any write to T3
   would be a spurious path not involving the DUT under test.

3. Adjacent string literal joins: Verilator v5.048 does not
   support C-style adjacent string concatenation in $display.
   All multi-line $display strings were merged to single
   string literals. A Python script was used to perform the
   merge to avoid manual error on the ~20 affected calls.

4. alc_comp forced to 3'd0 in all five tests after the
   prediction capture step. This suppresses allocation,
   which would otherwise overwrite the T3 isolation
   reference and T4 during misprediction tests, contaminating
   the readback.
