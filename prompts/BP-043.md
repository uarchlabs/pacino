## Hypothesis
The IA-generated CTR and USE tests in tb_tage.sv were
written against earlier versions of the planning documents.
The CTR table revision in BP-041 removed ~50 rows and
corrected X-entry handling. The USE table DIFF definition
correction means any test using pred_tkn vs alt_tkn for
DIFF is wrong. Some existing tests may now exercise
unreachable conditions or have incorrect expected values.

## Background
tage_cntrl_ctr_update_rules.md was significantly revised
in BP-041. Key changes:
- Table reduced from ~80 rows to ~30 rows.
- X entries for pred_diff, pT, aT expanded and
  unreachable rows removed.
- Constraint 1: UP=1 requires PT=pT (tautology). Rows
  violating this removed.
- Constraint 2: pCMP=aCMP=0 requires diff=0. diff=1 rows
  in groups 13a-d removed.
- Row 13e added: UP=0 when pCMP=aCMP=0 is ADR-001
  violation. No RTL action.
- HAND-FIX-003: u_both_t0 path corrected. u_resolved
  replaced with !u_mispredict.
- ADR-001 established: tage_using_primary=1 when BIM
  is sole provider.

tage_cntrl_use_update_rules.md corrections in session-045:
- DIFF redefined as tage_prm_tkn != tage_alt_tkn.
  Previous definition (pred_tkn vs alt_tkn) was wrong.
- TTM row added.
- Row priority note added.
