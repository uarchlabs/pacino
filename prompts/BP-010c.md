## Decisions made not explicit in the prompt
D1. All test addresses use bank=0 (idx[10]=0) to keep
    address calculations simple and verifiable.
D2. alc_comp=0 used in TC2 and TC4 to suppress allocation
    write when cond_mispredict=1 but alloc is not under test.
D3. Expected values derived from RTL behavior (bweb_n
    field-selective write) rather than from rule semantics.
D4. Staging always_ff added to testbench to work around a
    Verilator 5.020 --timing limitation: blocking assignments
    to struct-typed array elements in initial block coroutines
    do not set Verilator activity flags, so downstream always_
    comb is not re-evaluated before the next posedge. Driving
    DUT update inputs via NBA from always_ff ensures activity
    flags are set, triggering correct combo chain evaluation.
    This adds one posedge of latency to the update stimulus
    (two @(posedge clk) per test: first latches staging, second
    fires the bw_ram write). The tie-off initial block no longer
    drives tage_upd_val_u0 or tage_upd_inp_u0; the always_ff
    handles both default (zero) and active states.
