## Hypothesis

using_primary=1 the g_ctr_upd block drives the alternate

the strobe (ittage_cntrl.sv) or the table-select /

## Background
Table entries can be set directly by backdoor write to the
bw_ram memory inside each ittage_table instance. The tables
are instantiated in a generate loop in ittage.sv. Each
ittage_table holds two bw_ram instances: u_ram_s0 (slot 0)
and u_ram_s1 (slot 1). The memory array is .mem[bank][entry].
The fast-init initial block in ittage_table.sv shows the
path: u_ram_s0.mem[b][i] and u_ram_s1.mem[b][i].

Entry packing (per ittage_table.sv, bit 0 = LSB):
  VAL [0], CTR [3:1], USE [5:4], EPC [7:6],
  TGT [45:8], TAG [ALLOC_DATA_WIDTH-1:46].
For a prediction to hit an entry: VAL=1 and the stored TAG
equals that PC's per-table tag_hash at prediction time.

Address split (from the bw_ram port connection):
  bank  = idx[THIS_INDEX_BITS-1]
  entry = idx[THIS_INDEX_BITS-2:0]
where idx is the per-table index hash for the PC.

A genuine alternate provider requires two different tables
hitting the same prediction: seed a longer table (e.g. IT2)
and a shorter table (e.g. IT1) at their respective index
hashes for the same PC, both VAL=1 with matching tags. The
longer table is primary, the shorter is alternate.
