## Finding 1 -- RTL Bug HAND-FIX-003 (CLOSED)

### Location
tage_cntrl.sv, ctr_upd_comb block, u_both_t0 path.

### Symptom
Rows 13a and 13b of tage_ctr_test failed. T0 CTR
written with wrong value. 13a expected INC (1->2),
got DEC result. 13b expected DEC (1->0), got INC
result.

### Root Cause
RTL used u_resolved[s] as the INC/DEC condition for
the T0 CTR update. u_resolved is the branch outcome
(resolved_taken), not the BIM prediction correctness.

When prm_comp=0 and alt_comp=0 (both_t0 true), the
BIM is the sole predictor. The correct update condition
is whether the BIM prediction matched the outcome:
pred_tkn == resolved_taken, which is !u_mispredict[s].

### Rule Citation
tage_cntrl_ctr_update_rules.md rows 13a-d.
- 13a: pred_tkn=0, resolved=0 -> correct -> INC
- 13b: pred_tkn=0, resolved=1 -> wrong  -> DEC
- 13c: pred_tkn=1, resolved=0 -> wrong  -> DEC
- 13d: pred_tkn=1, resolved=1 -> correct -> INC

RTL used resolved_taken directly. Rows 13c and 13d
passed accidentally because resolved happened to
match the correct condition in those cases.

### Fix Applied
```
// Before (wrong):
if (u_resolved[s]) begin

// After (correct):
if (!u_mispredict[s]) begin
```

u_mispredict[s] is already assigned as
tage_upd_inp_u0[s].cond_mispredict which equals
pred_tkn != resolved_taken. No new signals required.

### Significance
First instance in this project where a manually
written test with hand-generated expected values
caught a real RTL bug. The three-layer methodology
(Jeff owns expected values, Claude Code owns task
infrastructure, no RTL in context during test
writing) worked as intended.

## Finding 3 -- Test State Contamination Across Rows

### Description
pred_meta fields set explicitly for rows 13 persisted
into rows 14-17. Specifically tage_prm_ctr and
tage_alt_ctr were set to t0ctr for rows 13 and not
restored before row 14.

### Fix
Explicit restore at the row 14 boundary:
```
pred_meta.tage_prm_ctr = pctr;
pred_meta.tage_alt_ctr = actr;
```

### Lesson
Any field in pred_meta or upd_data that departs from
the test-wide default must be explicitly restored
before the next row group. Do not rely on set_ctr_row
to restore fields it does not own.

## Finding 4 -- Cross-Block RAM State Contamination

### Description
Rows 1-12 leave RAM state dirty at the pcomp and
acomp addresses. Without reset_ctr_entries at the
top of the EN_ROW_13_18 block, rows 13 inherited
stale CTR values from prior DEC operations.

### Fix
Add reset_ctr_entries at the start of each ifdef
block, not just at the start of each row.

### Lesson
Each ifdef-gated block must be independently
runnable and must not depend on the prior block
having left RAM in any particular state.

## Finding 5 -- Diagnostic Methodology

### Description
Static code review of tage_cntrl.sv, tage_bim.sv,
and tage.sv was insufficient to identify the T0
write bug. Multiple incorrect hypotheses were
generated before runtime diagnostics were added.

### Resolution
Targeted $display at the write strobe (ctr_we_s0,
ram_addr_s0, ram_din_s0) in tage_bim.sv immediately
exposed din=0x0. This confirmed the write was firing
with wrong data rather than not firing at all, which
focused the search on the data path not the enable.

### Lesson
When static review fails after two attempts, add
runtime diagnostics before continuing. Display the
write enable, address, and data together in one
$display to distinguish between write-not-firing
and write-firing-with-wrong-data. These are
different bugs requiring different investigations.

## Finding 6 -- Assertion Infrastructure Established

### Description
Row 18 of tage_cntrl_ctr_update_rules.md defines
an unreachable condition: prm_comp=0 with alt_comp>0.
This cannot be tested as a normal update row. It is
covered by a runtime assertion instead.

### Implementation
New file: rtl/core/frontend/branch_predictor/tb/
tage_assert.sv

Assertion module bound to tage via SystemVerilog
bind. Checks at two boundary points per assertion:
- tage_pred_rdy_p2: prediction output boundary.
- tage_upd_val_u0: update input boundary.

Two assertions implemented:

Row 18 invariant: fires if prm_comp=0 and alt_comp>0.
```
$error("[TAGE_ASSERT][UPD] slot=%0d
  prm_comp=0 alt_comp>0 invalid", s);
```

ADR-001 invariant: fires if UP=0 when pCMP=aCMP=0.
```
$error("[ADR-001][TAGE_ASSERT][UPD] slot=%0d
  using_primary=0 when prm_comp=0 alt_comp=0", s);
```

Bracket prefix enables grep-based log filtering:
  grep "\[TAGE_ASSERT\]" sim.log

tage_assert.sv included in sim_tage_manual target.
Guarded with `ifndef SYNTHESIS.

### Bind Pattern
Following the OpenTitan prim_assert / tlul_assert
convention: assertions live in a separate module,
bound to the DUT via a bind file included only in
simulation builds. This keeps RTL clean and
assertions excluded from synthesis without
preprocessor guards in the RTL itself.
tage_assert.sv is the reference implementation for
this project. Same pattern applies to ittage_assert.sv
and bp_cluster_assert.sv as those modules mature.

### Error Numbering
BUG-NNN scheme adopted for bugs found by manual
tests and assertions:
- BUG-001: HAND-FIX-003 T0 CTR condition (Finding 1)
- BUG-002: reserved for row 18 assertion if fired
- BUG-003: reserved for ADR-001 assertion if fired

## Finding 8 -- CTR Update Rule Table Correction
##             and ADR-001

### Description
tage_cntrl_ctr_update_rules.md contained X (don't
care) entries for pred_diff, pT, and aT. Expanding
those X entries to explicit combinations exposed
two structural constraints that eliminated large
numbers of unreachable rows:

Constraint 1: UP=1 requires PT=pT (tautology).
When using_primary=1, pred_tkn is by definition
prm_tkn. A row with UP=1, PT=1, pT=0 is
structurally impossible. It is not an assertion
target -- it simply cannot occur. Invalid rows
removed from the table with no assertion needed.

Constraint 2: pCMP=aCMP=0 requires diff=0.
When both providers are BIM, prm_tkn and alt_tkn
are driven by the same BIM read (confirmed in
tage_cntrl.sv pred_logic defaults). diff=1 rows
in groups 13a-d are unreachable. Removed from
table.

### Row Count Reduction
Original rows 1-17 with X expansion: ~80 rows.
After applying constraints: significantly reduced.
Groups 1-2: 4 rows each -> 2 rows each.
Groups 3-6: 1 row each (no X to expand).
Groups 7-8: 4 rows each -> 2 rows each.
Groups 9-12: 1 row each.
Groups 13a-d: 8 rows each -> 1 row each plus
  1 invalid/assert row.
Groups 14-17: 4 rows each -> 2 rows each.

### ADR-001 Established
During row 13 analysis, the question arose whether
UP should be 0, 1, or don't care when pCMP=aCMP=0.

Decision: UP=1 is mandatory when BIM is sole
provider. Rationale: BIM is by definition the
primary provider when no tagged table hits. The
concept of primary vs alternate does not apply.
using_primary=0 is architecturally meaningless
and disallowed.

RTL confirmed: tage_cntrl.sv pred_logic sets
using_prm_p1 to 1'b1 by default. The UAON
override to 0 is gated on prm_comp != 0 and
cannot fire in the BIM case.

ADR-001 recorded in tage_cntrl_ctr_update_rules.md
as first project ADR. Convention established:
ADRs recorded in the planning document that
governs the decision, not in a separate directory.
Rationale: context minimization. Claude Code loads
the ADR automatically when it loads the rule doc.

ADR-001 enforced by assertion in tage_assert.sv
on both prediction and update boundaries.

Row 13e added to table to document the UP=0
invalid condition and cite the assertion:
  UP=0, pCMP=aCMP=0 -> ASSERT, no RTL action.
