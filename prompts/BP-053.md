## Hypothesis

ITTAGE allocation selects the correct candidate table at predict
time, asserts the allocation write only for that table at update
time, writes the correct pre-hashed index, and initializes the
allocated entry per the rules. Allocation has never been the
unit under test, so its conformance is assumed, not established.

## Background

Allocation fires on indir_mispredict when the provider is not
the longest table (IT5). Candidate selection runs at predict
time: scan tables longer than the provider for u_eff == 0, skip
consecutive tables, pick the shortest-history qualifying table,
capture ittage_alc_comp / ittage_alc_idx / ittage_alc_tag into
meta. ittage_alc_comp == 0 is the no-candidate sentinel (IT0
placeholder, never a valid target); it suppresses the write.
At update time the write strobe asserts on indir_mispredict AND
provider < IT5 AND alc_comp != 0, gated per table by alc_tbl_sel.
The allocated entry is CTR=0, USE=0, EPC=lcl_epoch, TAG=alc_tag,
TGT=resolved_target, VALID=1.

The authority is ittage_cntrl_alloc_rules.md.
