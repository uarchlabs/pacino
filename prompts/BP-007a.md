## Hypothesis
A single parameterized tage_table.sv module can serve all
five TAGE table slots (T0-T4) with per-instance parameter
assignment controlling entry geometry. Two bw_ram instances
per tage_table provide independent read and write ports for
dual prediction support (NUM_PRED_SLOTS=2 default).

## Background
TAGE is a tagged geometric history length branch predictor.
It has five tables: T0 is the base table (no tag, no valid,
no useful, 2b CTR). T1-T4 are tagged tables with entry
layout:

  N:8    7      6       5      4      3      2      1      0
  <tag>  EPC[1] EPC[0]  USE[1] USE[0] CTR[2] CTR[1] CTR[0] VALID

Each tage_table instance wraps two bw_ram instances, one per
prediction slot. RAM0 serves slot 0, RAM1 serves slot 1.
Selection is structural -- slot index is the RAM select.
There is no runtime bank selection mux.

Up to two components can have their CTR field updated in a
single update cycle (primary and alternative). Only the CTR
field supports dual-component update. All other field writes
target the primary component only.

The module gates its own update writes by comparing THIS_TABLE
to prm_tbl_sel_u0 (for primary) and alt_tbl_sel_u0 (for
alternative CTR).

NUM_PRED_SLOTS=2 is the project default. Generate blocks must
gate slot 1 logic on NUM_PRED_SLOTS==2.

## Binding Previous Decisions
- Package import order: bp_defines_pkg first, bp_structs_pkg
  second. Import at file scope before module declaration.
- bw_ram is the approved SRAM primitive for this project.
- Prediction and update paths are mutually exclusive by
  design. The prediction path is read-only. The update
  path is write-only. No read-modify-write cycles are
  required or permitted.
- NUM_PRED_SLOTS=2 is the default. Slot 1 logic gated by
  generate on NUM_PRED_SLOTS==2.

### Prediction path
- tage_pred_val_p0 enables the SRAM read for that slot.
- index_hash_p0 is the read address into the RAM.
- RAM read is registered. Output is valid at p1.
- hit_p1: tag compare of stored tag field against
  tag_hash_p0, gated with the stored valid bit.
  T0 has no tag -- hit_p1 is always 1 when
  tage_pred_val_p0 was asserted. This is a known gap
  (Known Gap 1 in tage_table_interfaces.md). Do not
  implement T0 special casing now. Use THIS_TAG_BITS==0
  as the condition if you need to stub it.
- taken_p1: MSB of the stored CTR field. Not gated by
  hit. Consumer ignores when hit_p1=0.
- cntrl_bits_p1: full CNTRL_BITS_WIDTH read data from
  the entry, zero-extended if THIS_CTR_WIDTH or
  THIS_USE_WIDTH is less than MAX_*.

### Update path
- tage_upd_val_u0 enables update processing.
- All write enables are further gated by THIS_TABLE
  compared to prm_tbl_sel_u0 or alt_tbl_sel_u0 as
  specified in tage_table_interfaces.md.
- prm_ctr_wr_u0 gated by prm_tbl_sel_u0==THIS_TABLE:
  writes prm_ctr_wd_u0 to CTR field at upd_index_u0.
- alt_ctr_wr_u0 gated by alt_tbl_sel_u0==THIS_TABLE:
  writes alt_ctr_wd_u0 to CTR field at upd_index_u0.
- use_wr_u0 gated by prm_tbl_sel_u0==THIS_TABLE:
  writes use_wd_u0 to USE field at upd_index_u0.
- epc_wr_u0 gated by prm_tbl_sel_u0==THIS_TABLE:
  writes epc_wd_u0 to EPC field at upd_index_u0.
- alc_wr_u0 gated by prm_tbl_sel_u0==THIS_TABLE:
  writes alc_wd_u0 (full ALLOC_DATA_WIDTH) to entry
  at upd_index_u0. This is a full entry write including
  tag and all control bits.
- When tbl_ri_active is asserted, tbl_ri_wr overrides
  all write enables, tbl_ri_wa overrides all addresses,
  tbl_ri_wd overrides all write data. tbl_ri_wr must be
  converted to active-low bw_ram write enable.

- Address width = THIS_INDEX_BITS.
- Use generate block gating RAM1 on NUM_PRED_SLOTS==2.

### Assumptions and Deviations

A1. ALLOC_DATA_WIDTH deviation from spec formula.
    Spec formula: CNTRL_BITS_WIDTH + THIS_TAG_BITS.
    CNTRL_BITS_WIDTH = MAX_CTR_WIDTH+MAX_USE_WIDTH+MAX_VAL_WIDTH
    = 3+2+1 = 6. Spec ALLOC_DATA_WIDTH = 6+8 = 14 (T1).
    Entry layout in tage_table_interfaces.md shows 16 bits:
    VALID+CTR+USE+EPC+TAG = 1+3+2+2+8. The 14b formula omits
    EPC. alc_wr_u0 is described as "full entry write including
    all control bits"; EPC is a control bit per the entry layout.
    Implementation uses:
    ALLOC_DATA_WIDTH = THIS_VAL_WIDTH + THIS_CTR_WIDTH
                     + THIS_USE_WIDTH + THIS_EPC_WIDTH
                     + THIS_TAG_BITS
    = 16 (T1). This is the only self-consistent definition.
    CNTRL_BITS_WIDTH follows the spec exactly (6 for T1) and
    governs only the cntrl_bits_p1 output port width (no EPC).

A2. T0 special-casing not implemented per spec (Known Gap 1).
    THIS_TAG_BITS==0 is not handled; hit_p1 would incorrectly
    use a 0-bit tag comparison. T0 requires separate treatment
    at the tage module level.

A3. NUM_PRED_SLOTS defaults to 2 in tage_table (project default
    for TAGE). The bp_defines_pkg package retains NUM_PRED_SLOTS=1
    as the cluster-wide default. The module parameter shadows
    the package parameter; -Wno-VARHIDDEN is applied in both
    sim and lint targets.

A4. BANKS=2 used in bw_ram instantiation with bank_addr=1'b0
    (always bank 0). This avoids the zero-width port edge case
    when BANKS=1 gives BANK_BITS=$clog2(1)=0. The behavioral
    model wastes bank 1; the macro substitution boundary is
    unchanged.
A5. Generate for-loop (for genvar gs=0; gs<NUM_PRED_SLOTS; gs++)
    is used instead of an if(NUM_PRED_SLOTS==2) block. This
    achieves the same effect: slot 1 logic exists only when
    NUM_PRED_SLOTS==2. Pattern is consistent with ubtb.sv.
