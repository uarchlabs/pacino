## Background
Bug 1 -- bank_addr strapped to 1'b0 (both modules):
bw_ram is a dual-bank bit-write SRAM. bank_addr selects
which of the two banks is accessed. The row address
within the selected bank is the lower bits of the full
index. For BANKS=2 and a THIS_INDEX_BITS-wide index:

  bank_addr = index[THIS_INDEX_BITS-1]
  row_addr  = index[THIS_INDEX_BITS-2:0]

Both tage_bim.sv and tage_table.sv strap bank_addr to
1'b0, making bank 1 dead storage. Read bw_ram.sv to
confirm the correct port semantics before fixing.

Bug 2 -- TBL_SEL_WIDTH default incorrect (tage_table
only):
tage_table.sv uses $clog2(TAGE_NUM_TABLES)+1 as the
default value for TBL_SEL_WIDTH. The correct default
is TAGE_TBL_SEL_WIDTH from bp_defines_pkg, consistent
with tage_bim.sv.

Bug 3 -- ram_din_s0/ram_din_s1 write data construction
uses if/else chain (tage_table only):
The write data bus for each slot is constructed using
a dependent if/else if chain. This prevents simultaneous
assertion of prm_ctr_we with use_we or epc_we from
producing correct write data -- only one field gets
written per cycle. The correct construction uses
independent per-field assignments so that any
combination of write enables produces correctly
assembled write data with all asserted fields in place
simultaneously. This applies to both ram_din_s0 and
ram_din_s1. tage_bim does not have this issue (CTR
only, no simultaneous field writes).

- TBL_SEL_WIDTH correct default is TAGE_TBL_SEL_WIDTH.
