## Hypothesis
tb_bp_history can be rewired to the BP-069 interface and its golden
fold corrected to the BP-071 geometry, with directed tests that
prove the DUT's incremental fold and rollback recompute are equal
to one independent reference for both the single-slot and the
dual-slot (num_branches=2) cases, and with sim_history and
cov_history green.

## Background
Two prior changes land in this testbench:

BP-069 reworked bp_history.sv to the module-owned pointer model:
ghist_ptr / phist_ptr are registered OUTPUTS, the live pointer
advances inside the module by num_branches, and rollback is by
checkpoint index (rollback_ckpt_idx). The old tb_bp_history drives
the now-removed input ports and no longer matches.

BP-071 corrected the fold geometry to the Xiangshan FoldedHistory
convention: newest bit inserted at the HIGH end of the compLen-wide
folded register; leaving bit removed at position (H-1) % W; one
fold definition shared by the incremental update and the rollback
recompute, with recompute position mapping posmap(i) = (i+W-1) % W.
The fold helpers were widened to 64b (SC ST3 has H=W=64). The DUT
is lint-clean and all non-history bpu targets are green.

The current tb_bp_history fails (waived) for one reason: its
testbench-local golden ref_fold still uses the OLD geometry
(pos = i % W). TC1-TC7 pass (short sequences, fold window never
fills); TC8 $fatal (full window, stale golden disagrees with the
corrected DUT). This task replaces that golden and extends the
suite.

bp_history_decisions.md and bp_history_interfaces.md are the
authority. bp_history.sv is read-only here (do not modify the RTL).
The hash-rule docs are reference for the fold bit order the tables
consume. The packages and Makefile are read-only reference.

BP-070 (the prior testbench attempt) is abandoned; do not resume
it. Its error was a golden built to mirror each DUT path separately;
this task uses ONE independent reference (requirement 2).

## Binding Previous Decisions
- Interface is the BP-069 target (bp_history_interfaces.md, Port
  List). Drive num_branches[1:0], pred_taken[1:0],
  pred_pc[VA_WIDTH-1:0][2], ckpt_wr_en, ckpt_wr_idx, rollback_valid,
  rollback_ckpt_idx. Read ghist_ptr / phist_ptr, ckpt_ghist_ptr /
  ckpt_phist_ptr, ghr_buf / phr_buf, folded as OUTPUTS. The
  testbench drives NO pointer.
- Fold geometry is the BP-071 Xiangshan convention: newest at the
  high end; leaving bit at (H-1) % W; recompute posmap(i) =
  (i+W-1) % W; 64b working width. The golden reference in this
  testbench MUST implement this geometry.
- The reference fold MUST be independent of the DUT (CLAUDE.md
  self-contained tests). Compute the expected fold from the
  observed ghr_buf contents by the geometry definition. Do NOT
  call, bind to, or copy the DUT's fold_step / fold_ghr. Do NOT
  build one reference per DUT path; build ONE reference and check
  both DUT paths against it.
- Pointer is module-owned, advances by num_branches, 0 holds
  (decisions 3.1). Checkpoint captures the pre-advance pointer
  (decisions 6; interface Checkpoint Timing). Rollback by index
  restores pointer and recomputes folds (decisions 2.2). Rollback
  wins same-cycle (decisions 4). Folds are stale, not cleared, in
  the rollback cycle (decisions 5).
- Dual-slot fold order is slot 0 then slot 1 (decisions 3.3).

## Specific Requirements
Read tb_bp_history.sv, bp_history.sv, bp_history_interfaces.md, and
bp_history_decisions.md before writing.

1. Rewire to the new interface. Stop driving the removed pointer
   and old rollback-pointer ports; read the pointer outputs; drive
   rollback via rollback_ckpt_idx. Port the surviving original
   checks. Report how many original checks survive, and name any
   that no longer apply (e.g. a check that drove the pointer
   directly) with the reason.

2. One independent reference fold (golden). Implement a
   testbench-local function fold_ref(ghr_value, ptr, H, W) that
   computes the fold by the BP-071 geometry directly from a buffer
   value: the H in-window bits folded into W, newest at the high
   end, position mapping posmap(i) = (i+W-1) % W. This is the only
   reference. It must not reference DUT internals. Replace the
   stale ref_fold (pos = i % W).

3. Single-slot equivalence. For directed bit patterns with
   num_branches=1, after each update assert DUT folded ==
   fold_ref(ghr_buf, ...) for every tagged table (TAGE T1-T4 idx/
   tag_fh1/tag_fh2, ITTAGE IT1-IT4 idx/tag_fh1/tag_fh2, SC ST1-ST3
   idx). Include sequences long enough to FILL and wrap each fold
   window (the case the old suite never reached -- the bug lived
   past TC7).

4. TD #74 -- dual-slot equivalence (centerpiece). For directed
   patterns with num_branches=2:
   - Drive pred_taken[1:0] / pred_pc[2] for one cycle.
   - Assert the DUT folded output equals fold_ref computed from the
     post-update ghr_buf for every table. Because fold_ref is the
     single definitional reference, this simultaneously checks the
     incremental two-step (slot 0 then slot 1) path.
   - Then assert rollback to a checkpoint and confirm the DUT
     recompute also equals fold_ref from the restored pointer.
     DUT-incremental == fold_ref AND DUT-recompute == fold_ref
     together prove incremental == recompute (the property BP-071
     verified offline; this proves it in simulation).
   - Include a case where slot 1 differs from slot 0 in direction,
     and a case where the +2 advance crosses the GHR_WIDTH boundary
     mid-bundle (pointer wrap).
   - Include SC ST3 (H=W=64) explicitly, since that is the case
     that forced the 64b widening and cannot be represented at 32b.

5. num_branches semantics: num_branches=0 holds pointer / ghr_buf /
   phr_buf / folded; =1 advances by 1; =2 advances by 2 and writes
   slot 0 at the pointer, slot 1 at pointer+1.

6. Checkpoint and rollback by index: assert ckpt_wr_en at a chosen
   ckpt_wr_idx; confirm ckpt_ghist_ptr / ckpt_phist_ptr report the
   pre-advance pointer. Advance, then rollback_valid with
   rollback_ckpt_idx at that slot; confirm the pointer outputs
   return to the checkpointed position and folded matches fold_ref
   from there.

7. Rollback-wins (G21): assert rollback_valid and num_branches=2 in
   the same cycle; confirm the prediction update and checkpoint
   write are dropped (pointer and folds reflect the rollback; the
   targeted ckpt slot is unchanged).

8. Stale fold (G22): in the rollback cycle confirm folded holds its
   prior value; the recomputed value appears the next cycle. This
   is a timing observation, not an error.

9. Self-checking, final PASS/FAIL summary with counts; basic sanity
   run under 10 seconds in Verilator.
